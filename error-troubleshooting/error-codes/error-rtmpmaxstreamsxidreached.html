Significado

Este error significa que la CDN ha rechazado una transmisión RTMP porque hay demasiadas transmisiones que utilizan el mismo ID de transmisión. Por lo tanto, se excede el número máximo de transmisiones por ID.

Más información

Esto puede significar:

  • Los flujos máximos por ID configurados en la CDN son demasiado bajos para aceptar todos los flujos solicitados
  • Otros trabajos de Zencoder ya se están transmitiendo a este CDN e ID de transmisión
  • Otro sistema fuera de Zencoder está utilizando la ID de transmisión de CDN
  • La CDN podría tener «' secuencias zombis», en las que una transmisión anterior no se desconectó correctamente y sigue contando hacia el límite máximo de transmisión para este ID.

Para evitar este problema, puede detener los otros trabajos que tienen transmisiones con esta ID de transmisión o usar otra ID de transmisión si la tiene. Para más información puede soporte de contacto con el ID de trabajo y nuestro equipo de ingenieros lo investigará.

¿Reintentar este trabajo?

Trabajos con este error no debe volver a intentarlo. Obtenga más información sobre cómo reintentar trabajos.
