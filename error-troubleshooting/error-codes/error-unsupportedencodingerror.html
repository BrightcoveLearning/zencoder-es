Significado

Este error ocurre cuando detectamos que el archivo enviado fue codificado de una manera que actualmente no podemos admitir.

Más información

  • Flip4Mac WMV en contenedor QuickTime: Las extensiones Flip4Mac para QuickTime codifican archivos WMV en contenedores QuickTime de una manera patentada que solo él puede decodificar.
  • Varias descripciones de muestra para un solo flujo: Algunos archivos QuickTime contienen una codificación de video patentada donde se colocan múltiples pistas de video diferentes en una secuencia, a menudo como resultado de cortar / editar.
  • Manifiesto / lista de reproducción de transmisión HTTP en vivo: Actualmente, no procesamos transmisión de video y el archivo enviado era un manifiesto para una transmisión.
  • Audio PCM multicanal con una profundidad de bits inferior a 8: Archivos de audio envolvente / multicanal que almacenan múltiples muestras en un solo byte.
  • Audio multicanal con más de 8 canales: Actualmente solo admitimos la mezcla automática de archivos multicanal de hasta 8 canales.
  • Audio LPCM en flujo SMPTE 302M: Actualmente no admitimos archivos MPEG con audio LPCM mezclado en una transmisión SMPTE 302M.

Soluciones alternativas

La mayoría de las veces, los archivos que encuentran este error se codificaron con codificaciones específicas de la aplicación o propietarias. A menudo, la única forma de procesarlos es cargarlos en QuickTime o el software que se utilizó para crearlos y exportarlos en una codificación estándar.

¿Reintentar este trabajo?

Trabajos con este error no debe volver a intentarlo. Obtenga más información sobre cómo reintentar trabajos.
