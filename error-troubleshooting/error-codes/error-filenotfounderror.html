Significado

Intentamos descargar un archivo en la dirección especificada, pero no había ningún archivo allí. Intentamos descargar desde la URL especificada por la configuración de "entrada" en su solicitud de API. Este error podría significar que hay un error tipográfico en la opción de entrada, o que el archivo ha sido eliminado, o un bucle de redireccionamiento, o algo más.

En términos técnicos, esto podría significar algunas cosas. En el caso de HTTP, esto significa que obtuvimos un código de respuesta HTTP> = 300 cuando intentamos descargar el archivo, excepto 403, que es un error diferente. Con FTP y FTPS, esto significa que recibimos un código de respuesta FTP 550, que indica que el archivo no estaba presente o no disponible. Con el resto de los protocolos, esto simplemente significa que no se encontró el archivo.

Soluciones alternativas

Asegúrese de que el archivo exista en la URL proporcionada y que el usuario con el que nos conectamos pueda acceder a él. Si no existe en esa ubicación, mueva el archivo antes de volver a intentarlo. Si existe en esa ubicación, verifique los permisos del archivo y si el usuario dado tiene acceso a la ruta del archivo.
