Significado

Este error significa que la CDN ha rechazado una transmisión RTMP porque el nombre de la transmisión ya está en uso por otra transmisión. En otras palabras, dos codificadores RTMP están intentando pasar al mismo punto de entrada CDN y el segundo se rechaza porque el primero ya está funcionando.

Más información

Esto puede significar:

  • Otro trabajo de Zencoder ya se está transmitiendo a este nombre de transmisión de CDN
  • El nombre del flujo de CDN está siendo utilizado por otro sistema fuera de Zencoder
  • Un trabajo anterior no se pudo desconectar de la CDN por algún motivo

Para evitar este problema, puede usar un nombre de flujo de CDN diferente o terminar cualquier trabajo anterior usando el mismo nombre de flujo. Si un trabajo no termina por alguna razón, soporte de contacto con el ID de trabajo y nuestro equipo de ingenieros lo investigará.

¿Reintentar este trabajo?

Trabajos con este error no debe volver a intentarlo. Obtenga más información sobre cómo reintentar trabajos.
