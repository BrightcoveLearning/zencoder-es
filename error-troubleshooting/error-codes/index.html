Códigos de error

Este tema proporciona un resumen de los códigos de estado HTTP para las solicitudes de la API de Zencoder y una índice de códigos de error devuelto por la API de Zencoder. Haga clic en cualquiera de los códigos de error para ver más detalles.

Códigos de estado HTTP

La siguiente tabla enumera el código de respuesta HTTP para las solicitudes de la API de Zencoder, en el caso de errores, indica la causa más probable, cuando sea posible.

Códigos de respuesta HTTP
Código de estado Explicación probable
200 OK Respuesta de éxito estándar para acciones que no crean un activo nuevo
201 CREATED Respuesta estándar para solicitudes que crean nuevos activos
204 NO CONTENT Respuesta estándar para una solicitud que no devuelve respuesta
400 BAD REQUEST JSON no válido en el cuerpo de la solicitud
401 UNAUTHORIZED Clave de API no válida o no válida
402 PAYMENT REQUIRED Se devuelve cuando una solicitud de API intenta desactivar el modo de integración de Zencoder https://app.zencoder.com/docs/api/accounts/integration en una cuenta donde no se ha configurado la facturación. Deberías ir a www.zencoder.com y elige un plan de pago
403 FORBIDDEN límite de velocidad para las solicitudes superadas o en respuesta a una solicitud de API de reintentación/reenvío de recursos que se han vuelto a intentar demasiadas veces
404 NOT FOUND URL de solicitud incorrecta o ID de trabajo antiguo o no válido
406 NOT ACCEPTIBLE Devuelto por intentar crear una credencial que no es de AWS a través de la API
409 CONFLICT Generalmente, solo cuando se intenta volver a intentar / volver a enviar un trabajo que es demasiado antiguo
422 UNPROCESSIBLE ENTITY Devuelto cuando una solicitud de API no se puede validar
500 INTERNAL SERVER ERROR O un problema temporal con Zencoder o algo mal con la solicitud
503 SERVICE UNAVAILABLE Zencoder no disponible temporalmente, probablemente durante el mantenimiento programado
504 GATEWAY TIMEOUT Devuelto si el servicio Zencoder no funciona
