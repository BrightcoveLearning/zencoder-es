PREGUNTAS MÁS FRECUENTES: Transferencias de archivos

Este tema responde preguntas sobre la transferencia de archivos a Zencoder para la codificación y a sus destinos para las salidas.

Preguntas más frecuentes

  • ¿Qué protocolos de transferencia admiten? HTTP? FTP?

    Entrada: admitimos HTTP, HTTPS, FTP, SFTP, Rackspace Cloud Files, Google Cloud Storage, Azure y Amazon S3.

    Salida: admitimos FTP, SFTP, Rackspace Cloud Files, Azure, Google Cloud Storage y Amazon S3. Actualmente no admitimos cargas HTTP generales.

    Usuarios de Cloud Files: Puede especificar la región a usar (DFW, ORD o Reino Unido) agregándola al protocolo, como cf+ord://username:api_key@container/object. Actualmente, las únicas regiones de Cloud Files compatibles son DFW, ORD y UK. La región DFW se utilizará de forma predeterminada.

    ¿Necesitas otro método de transferencia? Haznos saber.

  • ¿Dónde puedo colocar mis archivos originales para la transcodificación?

    Podemos extraer archivos desde cualquier lugar a través de HTTP, HTTPS, FTP o SFTP. También podemos extraer archivos de un contenedor de Azure, Google Cloud Storage o un depósito de S3, incluidos depósitos privados, y de Rackspace Cloud Files.

    Ejemplos:

    http://example.com/path/to/file.avi
http://s3.amazonaws.com/bucket/file.avi
s3://bucket/file.avi
gcs://bucket/file.avi
cf://username:api_key@container/file.avi
azure://account-name:account-key@container/file.avi
https://example.com/path/to/file.avi
ftp://example.com/path/to/file.avi
sftp://example.com/path/to/file.avi
    Nota sobre los puertos: si necesita incluir un puerto en la dirección del archivo, debe declararse al final del dominio en la URL, así: 
    sftp://example.com:8888/path/to/file.avi

  • ¿Puedes transferir archivos de forma segura?

    Seguro. Use SFTP o HTTPS e incluya un nombre de usuario y contraseña en la URL, como este:

    
https://user:pass@example.com/file.mp4
sftp://user:pass@example.com/file.mp4

  • ¿Puede extraer archivos de un depósito privado de S3, Google Cloud Storage o Cloud Files?

    Si. Puedes agregar acceso cartas credenciales a su cuenta de Zencoder, o conceder acceso de lectura al depósito para Zencoder.

  • ¿Existe algún límite de tamaño de archivo?

    No. No tenemos límites de tamaño de archivo y, de hecho, optamos por no cobrar más por videos de entrada grandes para alentar a los clientes a enviarnos sus mejores copias. Cuanto mayor sea la calidad de los videos que nos envíe, mayor será la calidad de salida que podemos brindarle.

    Sin embargo, hay un límite de 24 horas en la duración del video fuente.

  • ¿A dónde van los videos terminados?

    Tiene algunas opciones sobre dónde enviaremos los videos.

    Podemos enviar los archivos a un servidor FTP o SFTP, a un depósito de S3 o Google Cloud Storage, oa Rackspace Cloud Files. Ver el opción url en nuestros documentos API para más información.

    No olvides agregar cartas credenciales o dar acceso a Zencoder a su depósito si usa S3 o Google Cloud Storage.

    O puede especificar ningún destino de salida y guardaremos el archivo durante 24 horas. Le proporcionaremos una URL que puede usar para descargar el video. Eliminaremos el video después de 24 horas, así que no olvide descargarlo.

    A partir de la versión 2 de la API, si no podemos cargar archivos en su servidor, los cargaremos en S3 y le proporcionaremos una URL para todos los archivos.

  • ¿Cómo configuro los permisos de ACL si envío archivos codificados a S3?

    Utilice nuestra API. Consulte los documentos de la API en control de acceso para más información.

    • Transferencias de archivos

    Zencoder se ejecuta en instancias de Amazon EC2, por lo que existe una amplia gama de direcciones IP que usamos para la transcodificación de archivos y en vivo. Amazon agrega regularmente nuevos rangos a la lista, que se pueden encontrar a través de AWS API de rango de IP.

    Notificaciones

    Actualmente se envían notificaciones desde sg-77f03012, pero esto podría cambiar en cualquier momento. Para tener en cuenta esto, hemos agregado un X-Zencoder-Notification-Secret encabezado a todas las notificaciones entregadas. Esto se puede encontrar en el API panel de control, donde puede ver su secreto actual y generar uno nuevo si es necesario.

    Este encabezado se entrega con cada solicitud y se puede utilizar para garantizar que la notificación se envió desde su cuenta de Zencoder. Nota: Para mantener segura su notificación en secreto, asegúrese de enviar notificaciones a los puntos finales HTTPS.

    Nota: Los servidores pueden trasladarse a nuevos centros de datos y las direcciones IP pueden cambiar sin previo aviso.

  • ¿Puedo enviar un disco duro con mis videos a Zencoder?

    Si tiene una gran cantidad de videos para codificar pero no desea transferirlos a través de Internet, puede usar Amazon Importación / exportación de AWS. Enviará un disco duro con sus videos a Amazon, quien los cargará en S3 por usted.

    Amazon tiene instrucciones sencillas para preparar su disco para el envío, dónde enviarlo y más.
