Descripción general

Zencoder admite el procesamiento de archivos multimedia cifrados utilizando el paradigma de "cifrado en reposo"; recuperamos los archivos cifrados de su servidor y solo los desciframos durante el procesamiento. La versión descifrada del archivo se elimina inmediatamente después del procesamiento, manteniendo sus datos seguros.

Admitimos archivos cifrados AES-128 y AES-256 en los modos CBC y CTR. Los parámetros de descifrado deben incluirse con su solicitud de trabajo. El descifrado está habilitado cuando un descifrado_clave o decryption_key_url está especificado.

método_decifrado

decryption_method:String

Versiones de API: V1, V2

Padre: aporte

Defecto: aes-128-cbc (si descifrado_clave o decryption_key_url se establecen)

Valores válidos: "ninguno", "aes-128-cbc", "aes-128-ctr", "aes-256-cbc" o "aes-256-ctr"

Tipos de trabajo compatibles: vod

Ejemplo: aes-128-cbc

Descripción:

Configure el algoritmo de descifrado que se utilizará para descifrar.

  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_method": "aes-128-cbc"
  }

descifrado_clave

decryption_key:String

Versiones de API: V1, V2

Padre: aporte

Valores válidos: Una cadena hexadecimal de 16 octetos (32 caracteres de longitud, prefijo "0x" opcional)

Tipos de trabajo compatibles: vod

Ejemplo: 1234567890abcdef1234567890abcdef

Descripción:

Configure la clave de descifrado que se utilizará para una entrada cifrada.

  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_key": "1234567890abcdef1234567890abcdef"
  }

decryption_key_url

decryption_key_url:String

Versiones de API: V1, V2

Padre: aporte

Valores válidos: Una URL a un archivo que contiene una clave binaria de 16 bytes

Tipos de trabajo compatibles: vod

Ejemplo: http://example.com/encryption.key

Descripción:

La URL de un archivo de clave de descifrado que se utilizará, con una clave en formato binario (16 bytes en total). Esto se ignora si se especifica una clave_decifrado. 
  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_key_url": "http://example.com/encryption.key"
  }

contraseña_decifrado

decryption_password:String

Versiones de API: V1, V2

Padre: aporte

Valores válidos: Una cuerda

Tipos de trabajo compatibles: vod

Ejemplo: contraseña-supersecreta

Descripción:

La contraseña utilizada en combinación con la clave para descifrar el archivo de entrada.

  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_password": "super-secret-password"
  }
