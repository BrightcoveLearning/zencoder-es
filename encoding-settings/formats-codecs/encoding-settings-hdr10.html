    Configuración HDR

    Este tema contiene detalles sobre la configuración de codificación relacionada con las salidas de vídeo HDR.

    Resumen

    Para obtener más información sobre los formatos HDR compatibles con Zencoder y las recomendaciones sobre los parámetros de formato HDR, consulte la Guía de vídeo HDR.

    preserve_hdr_metadata

    master_display:Boolean

    Versiones de API: V2

    Principal: salidas

    Predeterminado: false

    Tipos de trabajo compatibles: vod

    Descripción:

    Cuando se establece en true, este parámetro indica a Zencoder que importe (la mayoría o todos) metadatos relacionados con HDR del archivo de entrada y pasarlos a la salida.

    Por ejemplo, la transcodificación de flujos de vídeo codificados en HEVC en formatos HDR10, PQ10 o HLG10 se puede lograr mediante la siguiente solicitud de trabajo: 

    {
  "input": "s3://mybucket/hdr_master.mp4",
  "outputs": [
    {
     "video_codec": "hevc",
     "video_codec_profile": "main10",
     "video_bit_depth": 10,
     "preserve_hdr_metadata": true
    }
  ]
}

    Este parámetro también se puede utilizar en combinación con otros parámetros, especificando directamente los metadatos relacionados con el HDR. Por ejemplo, a continuación, la solicitud de trabajo JSON muestra cómo el preserve_hdr_metadata parámetro también se puede utilizar para transportar master display y content light level de secuencias DolbyVision 8.1 codificadas en HEVC, mientras que los metadatos dinámicos (RPU archivo) de DolbyVision se comunican por separado mediante el parámetro dolby_vision_rpu_url:

    {
  "input": "s3://mybucket/dv81_master.mp4",
  "outputs": [
    {
     "video_codec": "hevc",
     "video_codec_profile": "main10",
     "video_bit_depth": 10,
     "hdr_format": "dv8.1",
     "preserve_hdr_metadata": true,
     "dolby_vision_rpu_url": "s3://mubucket/dv81_master_rpu.rpu"
    }
  ]
}

    formato hdr_

    hdr_format:String

    Versiones de API: V2

    Padres:outputsdynamic_profile_options

    Valores válidos: hdr10 , pq , hlg , dv5 , dv8.1

    Predeterminado: ninguno

    Tipos de trabajo compatibles: vod

    Descripción:

    Especifica el tipo de formato HDR que se está utilizando para producir salidas.

    Este parámetro debe utilizarse en los casos en que los archivos de entrada no contienen metadatos relacionados con formato HDR y, por lo tanto, tanto el tipo de formato como todos los metadatos asociados deben especificarse directamente.

    Por ejemplo, un contenido graduado de color PQ proporcionado como MFX codificado J2K se puede convertir a flujo de salida PQ10 mediante la siguiente solicitud de trabajo:

    {
  "input": "s3://mybucket/j2k_master.mfx",
  "outputs": [
    {
     "video_codec": "hevc",
     "video_codec_profile": "main10",
     "video_bit_depth": 10,
     "hdr_format": "pq"
    }
   ]
}

    O. si el mismo maestro J2K MXF representa contenido HDR10, y viene con archivos adicionales que describen su pantalla maestra y parámetros de metadatos de nivel de luz de contenido , entonces la codificación de la misma en el flujo HDR10 HEVC se puede lograr mediante el siguiente trabajo solicitud:

    {
  "input": "s3://mybucket/j2k_master.mfx",
  "outputs": [
    {
      "video_codec": "hevc",
      "video_codec_profile": "main10",
      "video_bit_depth": 10,
      "hdr_format": "hdr10",
      "master_display": {
        "red": {"x": 0.708, "y": 0.292},
        "green": {"x": 0.17, "y": 0.797},
        "blue": {"x": 0.131, "y": 0.046},
        "white_point": {"x": 0.31273,"y": 0.329},
        "luminance": {"max": 1000,"min": 0.0001}
      },
      "max_content_light_level": 1000,
      "max_frame_average_light_level": 180
    }
  ]
}

    master_display

    master_display:Object

    Versiones de API: V2

    Padres:outputsdynamic_profile_options

    Predeterminado: ninguno

    Tipos de trabajo compatibles: vod

    Descripción:

    Estructura que describe el volumen de color (las primarias de color, el punto blanco y el rango de luminancia) de la pantalla que se utilizó para dominar el contenido de vídeo. Los valores específicos utilizados en esta estructura están definidos por el estándar SMPTE ST 2086 .

    Los metadatos de visualización maestra son obligatorios y deben llevarse a cabo en las salidas al codificar vídeos en formatos HDR-10 y DolbyVision 8.1.

    El siguiente ejemplo muestra cómo se pueden especificar los metadatos de visualización maestra:

    {
  "input": "s3://mybucket/j2k_master.mfx",
  "outputs": [
    {
      "video_codec": "hevc",
      "video_codec_profile": "main10",
      "video_bit_depth": 10,
      "hdr_format": "hdr10",
      "master_display": {
        "red": {"x": 0.708, "y": 0.292},
        "green": {"x": 0.17, "y": 0.797},
        "blue": {"x": 0.131, "y": 0.046},
        "white_point": {"x": 0.31273,"y": 0.329},
        "luminance": {"max": 1000,"min": 0.0001}
      },
      "max_content_light_level": 1000,
      "max_frame_average_light_level": 180
    }
  ]
}

    max_content_light_level

    max_content_light_level:Number

    Versiones de API: V2

    Padres:outputsdynamic_profile_options

    Predeterminado: ninguno

    Valores válidos:0-10000

    Tipos de trabajo compatibles: vod

    Ejemplo:1000

    Descripción:

    Describe el nivel máximo de luz en píxel (y canal rojo, verde y azul) en toda la secuencia de vídeo. Expresado en candellas por metro cuadrado. Este valor se puede establecer 0, lo que implica que dicho enlace es desconocido o no impuesto. Este parámetro, también conocido como MaxCLL está definido por el estándar CAE 861.3 .

    El siguiente ejemplo muestra cómo se pueden especificar los metadatos max_content_light_level y max_frame_average_light_level:

    {
  "input": "s3://mybucket/j2k_master.mfx",
  "outputs": [
    {
      "video_codec": "hevc",
      "video_codec_profile": "main10",
      "video_bit_depth": 10,
      "hdr_format": "hdr10",
      "master_display": {
        "red": {"x": 0.708, "y": 0.292},
        "green": {"x": 0.17, "y": 0.797},
        "blue": {"x": 0.131, "y": 0.046},
        "white_point": {"x": 0.31273,"y": 0.329},
        "luminance": {"max": 1000,"min": 0.0001}
      },
      "max_content_light_level": 1000,
      "max_frame_average_light_level": 180
    }
  ]
}

    max_frame_average_light_level

    max_frame_average_light_level:Number

    Versiones de API: V2

    Padres:outputsdynamic_profile_options

    Predeterminado: ninguno

    Valores válidos:5.0-10000.0

    Tipos de trabajo compatibles: vod

    Ejemplo:180

    Descripción:

    Describe el nivel máximo de luz promedio de fotogramas (y máximo en rojo, verde y azul) en toda la secuencia de vídeo. Expresado en candellas por metro cuadrado. Este valor se puede establecer 0, lo que implica que dicho enlace es desconocido o no impuesto. Este parámetro, también conocido como MaxFALL está definido por el estándar CAE 861.3 .

    Tanto los parámetros max_content_light_level como max_frame_average_light_level son obligatorios y deben incluirse en las salidas al codificar vídeos en formato HDR-10.

      "video_bit_depth": 10,
      "hdr_format": "hdr10",
      "master_display": {
        "red": {"x": 0.708, "y": 0.292},
        "green": {"x": 0.17, "y": 0.797},
        "blue": {"x": 0.131, "y": 0.046},

    video_color_range

    video_color_range:String

    Versiones de API: V2

    Padres:outputsdynamic_profile_options

    Valores válidos: limited , full

    Predeterminado: ninguno

    Tipos de trabajo compatibles: vod

    Descripción:

    Indica el rango dinámico de los valores de píxel. Por ejemplo, para píxeles de 8 bits, limited implica que los valores de píxel Y (luma) permanecerán en el rango de 16 a 235, mientras que full implica que los valores de píxel Y (luma) estarán en el rango de 0 a 255.

    La gran mayoría del contenido se produce actualmente y codifica utilizando limited píxeles. Sin embargo, a veces también se pueden utilizar píxeles de fullrango. Por ejemplo, se utilizan píxeles de rango completo de 10 bits de ancho en el formato DolbyVision 5.

    Ejemplo de uso: 

    {
  "input": "s3://mybucket/dv5_master.mp4",
  "outputs": [
   {
    "video_codec": "hevc",
    "video_codec_profile": "main10",
    "video_bit_depth": 10,
    "hdr_format": "dv5",
    "video_color_range": "full",
    "dolby_vision_rpu_url": "s3://mubucket/dv5_master_rpu.rpu"
   }
  ]
}
