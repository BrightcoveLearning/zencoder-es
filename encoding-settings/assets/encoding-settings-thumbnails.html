    Configuración de miniaturas

    Este tema contiene detalles sobre la configuración de codificación relacionada con las miniaturas de vídeo.

    miniaturas

    Miniaturas: matriz o hash

    Versiones de API: V1, V2

    Principal: salidas

    Tipos de trabajo compatibles: Todos

    Ejemplo:

    • «miniaturas»: [{}]
    • «miniaturas»: {}

    Descripción:

    Para cada salida, puede pedir una o más miniaturas.

    Cada salida puede tener sus propios conjuntos de miniaturas con un tamaño único, base_url y access_control conjunto.

    Se pueden crear varios conjuntos de miniaturas para una única salida pasando varios hash de miniaturas dentro de una matriz. Si las opciones de miniatura se pasan en una matriz, se debe establecer una etiqueta para establecer.

    Establezca S3 access_control con las mismas opciones que utiliza para los vídeos de salida.

    Las miniaturas están limitadas a 100 por conjunto. Si se generaran más miniaturas dadas las opciones solicitadas, se añadirá una advertencia a la salida y la creación de miniaturas se detendrá cuando se alcance el límite.

    Nota: No existe tal cosa como un trabajo de sólo miniatura. Especificar una salida con solo miniaturas también creará un mp4 facturable de longitud completa con nuestra configuración de transcodificación predeterminada y lo cargará al almacenamiento temporal.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: [
        {
          «etiqueta»: «primero»,
          «número»: 10
        },
        {
          «etiqueta»: «segundo»,
          «interval_in_frames»: 5 000
        }
      ]
    }
  ]
}

    Consulte también: etiqueta

          «etiqueta»: «cartel»
        },
        {
          «etiqueta»: «clips»
        }
      ]
    }
  ]
}

    número

    Número:ENTERO

    Versiones de API: V1, V2

    Principal: salidas/miniaturas

    Valores válidos: Un entero positivo

    Tipos de trabajo compatibles: Todos

    Ejemplo: 3

    Descripción:

    Varias miniaturas para capturar. Zencoder capturará miniaturas uniformemente espaciadas a lo largo de toda la duración del archivo. Por lo tanto, si pide 1 miniatura, estará cerca del centro del archivo. La especificación de 3 miniaturas en un vídeo de 8 minutos dará como resultado imágenes en miniatura aproximadamente 2, 4 y 6 minutos.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: {
        «número»: 3
      }
    }
  ]
}

    start_at_first_frame

    INICIO_AT_FIRST_FRAME:booleano

    Versiones de API: V1, V2

    Principal: salidas/miniaturas

    Predeterminado: false

    Valores válidos: true o false

    Tipos de trabajo compatibles: Todos

    Ejemplo: true

    Descripción:

    Cuando se utiliza la opción numérica para la generación de miniaturas, esto comenzará a tomar miniaturas con el primer fotograma.

    Por ejemplo, cuando se utiliza si el vídeo tiene una duración de 5 segundos y solicita 5 miniaturas utilizando la opción de número , esto creará miniaturas a 0s, 1s, 2s, 3s y 4s. Normalmente agarraría miniaturas a 0.83s, 1,66, 2.49, 3.32 y 4,15 s.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: {
        «número»: 5,
        «start_at_first_frame»: true
      }
    }
  ]
}

    intervalo

    Intervalo:flotador

    Versiones de API: V1, V2

    Principal: salidas/miniaturas

    Tipos de trabajo compatibles: Todos

    Ejemplo: 60

    Descripción:

    Tome miniaturas en un intervalo uniforme, en segundos. Zencoder devolverá una miniatura por cada N segundos del archivo. Por lo tanto, si elige un intervalo de 60, y su archivo de entrada dura 12 minutos, obtendrá 12 miniaturas, cada una en el minuto.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: {
        «intervalo»: 60
      }
    }
  ]
}

    Consulte también: interval_in_frames

    interval_in_frames

    INTERVAL_IN_FRAMES:flotador

    Versiones de API: V1, V2

    Principal: salidas/miniaturas

    Tipos de trabajo compatibles: Todos

    Ejemplo: 120

    Descripción:

    Tome miniaturas en un intervalo uniforme, en fotogramas. Zencoder devolverá una miniatura por cada N fotogramas del archivo. Por lo tanto, si elige un intervalo de 120, y su archivo de entrada es de 29.97 fotogramas por segundo, obtendrá miniaturas aproximadamente cada 4 segundos.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: {
        «interval_in_frames»: 120
      }
    }
  ]
}

    veces

    veces:matriz

    Versiones de API: V1, V2

    Principal: salidas/miniaturas

    Tipos de trabajo compatibles: Todos

    Ejemplo: [0, 30, 60, 90]

    Descripción:

    Una matriz de veces, en segundos, en las que tomar una miniatura. Los decimales son válidos. Por lo tanto, si se ajustan los tiempos a [12.5, 25], se obtendrían dos miniaturas, una a 12,5 segundos y otra a los 25 segundos. Este valor debe ser una matriz de números no negativos.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: {
        «veces»: [0, 30, 60, 90]
      }
    }
  ]
}

    aspect_mode

    aspect_mode:cadena

    Versiones de API: V1, V2

    Principal: salidas/miniaturas

    Predeterminado: conservar

    Valores válidos: conservar, estirar, recortar o pad

    Tipos de trabajo compatibles: Todos

    Ejemplo: pad

    Descripción:

    El modo de aspecto que se va a utilizar al crear miniaturas.

    • preservar: De forma predeterminada, Zencoder conservará la relación de aspecto del archivo de vídeo de salida, por lo que si envía contenido en pantalla panorámica y solicita una resolución estándar, el archivo de miniaturas mantendrá la relación de aspecto de pantalla ancha y se ajustará al tamaño solicitado.
    • estiramiento: Cuando el modo de aspecto es «estirar», la miniatura coincidirá exactamente con el ancho y la altura solicitados, incluso si distorsiona la imagen.
    • cultivo: Esta opción le indica a Zencoder que «acerque» la miniatura para que coincida con el tamaño solicitado, recortando píxeles desde la parte superior/inferior o izquierda/derecha.
    • almohadilla: La opción del pad le indica a Zencoder que buzón la miniatura coincida con el tamaño de fotograma solicitado. Utilice esta opción para mantener la relación de aspecto del vídeo de salida, pero siempre obtener miniaturas del mismo tamaño.

    Nota: Debido a las restricciones de los algoritmos de escala de vídeo, las dimensiones de las miniaturas serán un número par. Si solicita miniaturas de 100x75, por ejemplo, obtendrá miniaturas de 100x74.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: {
        «aspect_mode»: «pad»,
        « ancho»: 200,
        « altura»: 200
      }
    }
  ]
}

    size

    Tamaño: cadena.

    Versiones de API: V1, V2

    Principal: salidas/miniaturas

    Tipos de trabajo compatibles: Todos

    Ejemplo: 400 x 300

    Descripción:

    Una resolución objetivo para las miniaturas, como «160×120″. Si no se proporciona ningún tamaño, las miniaturas tendrán el mismo tamaño que el vídeo de salida. Por defecto, conservaremos la relación de aspecto, por lo que si la relación de aspecto de este parámetro de tamaño no coincide con la relación de aspecto de la película, es posible que el archivo resultante no coincida exactamente con este tamaño. Si Stretch se establece en true para el archivo de salida correspondiente, esta miniatura también se estira para rellenar la resolución.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: {
        «tamaño»: «400x300"
      }
    }
  ]
}

    base_url

    URL_base:cadena

    Versiones de API: V1, V2

    Principal: salidas/miniaturas

    Valores válidos: Una URL válida de S3, Cloud Files, GCS, FTP, FTPS o SFTP, menos nombre de archivo. Formato: s3://[nombre del bucket]/[ruta] o ftp://[user]:[password]@[ftp.url]/[path]

    Tipos de trabajo compatibles: Todos

    Ejemplo:

    • s3: //mi-salida-cubo
    • cf: //nombre_usuario:api_key @container /ruta/
    • ftp://user:password@ftp.example.com/path/to

    Descripción:

    Destino de salida para miniaturas. Si base_url está en blanco, almacenaremos miniaturas en nuestro cubo Zencoder S3. Los archivos almacenados en el depósito de Zencoder S3 estarán disponibles después de 24 horas. Tenga en cuenta que los nombres de archivo no son únicos entre salidas (por ejemplo, frame_0000.png), por lo que el destino debe ser un directorio único o prefijo clave para evitar sobrescribir archivos.

    Usuarios FTP: intentamos escribir desde la raíz de su servidor, así que use una ruta absoluta para su URL para asegurarse de que podemos escribir en su servidor correctamente.

    Notas sobre Credenciales:

    Si proporciona credenciales a través de base_url, estarán presentes en las URL que aparecen en las solicitudes a los detalles del trabajo, detalles de salida, etc. Si las credenciales se almacenan con Zencoder, no aparecerán en las direcciones URL de esas respuestas.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: {
        «base_url»: «s3: //mi-salida-cubo/»
      }
    }
  ]
}

    Consulte también: prefijo y nombre de archivo

    prefijo

    prefijo:cadena

    Versiones de API: V1, V2

    Principal: salidas/miniaturas

    Valor predeterminado: marco

    Tipos de trabajo compatibles: Todos

    Ejemplo:

    • pulgares
    • video_thumbnails

    Descripción:

    Los archivos en miniatura se exportarán con nombres de archivo secuenciales. Sin embargo, puede especificar un prefijo personalizado para los archivos en miniatura. Si no se especifica ningún prefijo, usaremos frame. Por ejemplo, si utiliza el prefijo 'personalizado', los archivos exportados se denominarían custom_0000.png, custom_0001.png, etc. Sin un prefijo personalizado, se llamarían frame_0000.png, frame_0001.png, etc.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: {
        «prefijo»: «pulgares»
      }
    }
  ]
}

    Consulte también: base_url y nombre de archivo

    filename

    nombreDeArchivo:Cadena

    Versiones de API: V1, V2

    Principal: salidas/miniaturas

    Valor predeterminado: marco

    Tipos de trabajo compatibles: Todos

    Ejemplo:

    • miniatura _ {{number}}
    • {{number}} _ {{width}} x {{height}} -miniatura

    Descripción:

    A los archivos en miniatura se les puede dar un nombre de archivo basado en varios atributos. Los atributos incluyen número, número relleno, ancho, alto y tamaño. No es necesario añadir una extensión de archivo, ya que será añadida por Zencoder en función de la opción «formato».

    número o padded-number debe ser utilizado, todos los demás son opcionales. padded-number utiliza 4 puntos (0000, 0001, etc.).

    altura, ancho y tamaño se basarán en el tamaño real de la miniatura generada.

    Ejemplo: _x-miniatura

    Los caracteres válidos incluyen letras, números, guiones, guiones bajos y los valores interpolados.

    Tenga en cuenta que los caracteres especiales no son compatibles con los nombres de archivo, incluso si están codificados en URL.

    Nota: Si especifica un nombre de archivo, se ignorará cualquier prefijo especificado.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: {
        «filename»: «_x-thumbnail»
      }
    }
  ]
}

    Descripción:

    Zencoder admite la configuración de un subconjunto limitado de estos encabezados: Control de caché, Disposición de contenido, Codificación de contenido, Tipo de contenido, Expires, x-amz-acl, x-amz-storage-class, x-amz-server-side-cifrado y x-amz-meta-*.

    Los encabezados se ignorarán cuando no se especifique una ubicación de salida.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: {
        «encabezados»: {
          «Content-Type»: «binario/octet-stream»,
          «x-amz-acl»: «public-lectura-escritura»
        }
      }
    }
  ]
}

    credenciales

    Credenciales:cadena

    Versiones de API: V2

    Principal: salidas/miniaturas

    Tipos de trabajo compatibles: Todos

    Ejemplo: ftp_dev_server

    Descripción:

    Hace referencia al apodo de las credenciales guardadas que se utilizarán para la transferencia, que se administran en la sección Credenciales de cuenta.

    NOTA: Las credenciales de los servicios de distribución son sólo para vídeos de salida.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: {
        «base_url»: "ftp://ftp.example.com/thumbnails/ «,
        «credenciales»: «ftp_dev_server»
      }
    }
  ]
}

    Ver también: base_url

    parallel_upload_limit

    Parallel_upload_limit:entero

    Versiones de API: V1, V2

    Predeterminado: 30 para S3, 10 para todos los demás destinos.

    Valores válidos: Un entero entre 1 y 30

    Tipos de trabajo compatibles: Todos

    Ejemplo: 5

    Descripción:

    El número máximo de cargas simultáneas realizadas al cargar miniaturas.

    Esto puede acelerar los tiempos de transferencia, dependiendo del ancho de banda del servidor remoto. Tenga en cuenta que más conexiones pueden colocar una carga más pesada en el servidor. Si tiene problemas con los tiempos de espera de carga o desea evitar que Zencoder use demasiado ancho de banda al cargar archivos, configure esto en 1.

    {
  «input»: «s3: //zencodertesting/test.mov»,
  «productos»: [
    {
      «miniaturas»: {
        «base_url»: "ftp://ftp.example.com/thumbnails/ «,
        «parallel_upload_limit»: 5
      }
    }
  ]
}
