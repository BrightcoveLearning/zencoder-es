    Configuración de marcas de agua

    Este tema contiene detalles sobre la configuración de codificación relacionada con las marcas de agua de vídeo.

    marcas de agua

    marcas de agua:matriz o hash

    Versiones de API: V1, V2

    Principal: salidas OR dynamic_profile_options

    Valores válidos: Una matriz o hash de la configuración de marca de agua

    Tipos de trabajo compatibles: Todos

    Descripción:

    Puede agregar una o más marcas de agua a un vídeo de salida utilizando nuestra API de marca de agua. Zencoder admite hasta 4 archivos de marca de agua por salida en formato JPEG, BMP o PNG.

    • GIF, imágenes de marca de agua RGB+alfa de 32 bits y RGB+alfa de 64 bits no son compatibles.
    • Las marcas de agua que se extienden más allá de los bordes del vídeo no se aplicarán. Se generará una advertencia en la salida si esto ocurre.
    • Al usar el códec VP6 hay algunas limitaciones adicionales. Solo puede aplicar una marca de agua al vídeo (si se pasan varias marcas de agua, solo se usará la primera). Además, no se admite la escala de marca de agua y la imagen de marca de agua debe ser un archivo PNG.
      {
    «input»: «s3: //zencodertesting/test.mov»,
    «productos»: [
      {
        «marcas de agua»: [
          {
            «url»: «s3: //bucket/watermark_file.png»,
            «x»: «20",
            «y»: "-10% «,
            « ancho»: 32,
            « altura»: 24
          }
        ]
      }
    ]
  }

    url

    URL:cadena

    Versiones de API: V1, V2

    Principal: salidas/marcas de agua

    Valores válidos: Una dirección URL válida para un archivo de imagen.

    Tipos de trabajo compatibles: Todos

    Ejemplo:

    • https://s3.amazonaws.com/bucket/img.png
    • ftp://user:pass@example.com/path/to/watermark.jpg

    Descripción:

    La dirección URL de un archivo de imagen remota que se va a utilizar como marca de agua. Utilice la sintaxis de la URL de entrada . Soporta S3, GCS, Cloud Files, HTTP/S, FTP y SFTP, con o sin autenticación.

    Zencoder admite archivos de marca de agua en formato GIF, JPEG, BMP o PNG.

    Para marcas de agua transparentes, utilice PNG con transparencia alfa o índice.

    Formatos PNG admitidos

    • RGB de 8 bits
    • RGB+alfa de 8 bits
    • Escala de grises de 8 bits
    • Indexado de 8 bits
    • 8 bits+Transparencia
    • RGB de 16 bits
    • Escala de grises de 16 bits

    Formatos PNG no admitidos

    • Escala de gris+Alfa de 8 bits
    • Escala de gris+Alfa de 16 bits
    • RGB+alfa de 16 bits
      {
    «input»: «s3: //zencodertesting/test.mov»,
    «productos»: [
      {
        «marcas de agua»: {
          «url»: "https://s3.amazonaws.com/bucket/img.png»
        }
      }
    ]
  }

    x

    X: cadena o número

    Versiones de API: V1, V2

    Principal: salidas/marcas de agua

    Predeterminado ፦10

    Tipos de trabajo compatibles: Todos

    Ejemplo:

    • -0
    • 320
    • centrado

    Descripción:

    Dónde colocar la marca de agua dentro del vídeo, en el eje x (izquierda/derecha). Puede ser un número de píxeles (por ejemplo, 100 o -20), un porcentaje del ancho del vídeo (por ejemplo, 25% o -5%), o «centrado» para alinear la marca de agua con el centro del vídeo. Utilice un número positivo para colocar en relación con el lado izquierdo del vídeo y un número negativo para colocarlo en relación con el lado derecho del vídeo. Utilice «-0" (como una cuerda) para bloquear hacia el lado derecho.

    De forma predeterminada, la posición se basa en el área de contenido visible, sin incluir ningún relleno. Utilice el origen para establecer la posición de la marca de agua en función de la salida completa.

      {
    «input»: «s3: //zencodertesting/test.mov»,
    «productos»: [
      {
        «marcas de agua»: {
          «x»: 320
        }
      }
    ]
  }

    Ver también: y y origen

    anchura

    ancho:número o cadena

    Versiones de API: V1, V2

    Principal: salidas/marcas de agua

    Predeterminado: Escale hasta la altura o el ancho original de la imagen.

    Valores válidos: Un número positivo o un porcentaje

    Tipos de trabajo compatibles: Todos

    Descripción:

    El ancho de la marca de agua, expresado como un número de píxeles (por ejemplo, 64) o como porcentaje del ancho del vídeo (por ejemplo, 10%). Si se proporciona altura , pero no ancho, la imagen de marca de agua se escalará proporcionalmente.

      {
    «input»: «s3: //zencodertesting/test.mov»,
    «productos»: [
      {
        «marcas de agua»: {
          « ancho»: 100
        }
      }
    ]
  }

    Consulte también: altura

    altura:número o cadena

    Versiones de API: V1, V2

    Principal: salidas/marcas de agua

    Valores válidos: Un número positivo o un porcentaje

    Tipos de trabajo compatibles: Todos

    Descripción:

      {

    Ver también: ancho

    origen

    Origen:Cadena

    Versiones de API: V1, V2

    Principal: salidas/marcas de agua

    Predeterminado: contenido

    Valores válidos: contenido o marco

    Tipos de trabajo compatibles: Todos

    Ejemplo: marco

    Descripción:

    La parte del vídeo en la que basar el posicionamiento de la marca de agua. Esto solo afecta a los trabajos donde aspect_mode es 'pad'.

    • 'frame 'basa la colocación en la resolución completa de la salida, incluyendo cualquier relleno.
    • 'content' basa la ubicación en el área de contenido visible, sin incluir el relleno.
      {
    «input»: «s3: //zencodertesting/test.mov»,
    «productos»: [
      {
        «marcas de agua»: {
          «origen»: «contenido»
        }
      }
    ]
  }

    Consulte también: x , y y aspect_mode

    opacidad

    Opacidad:flotador.

    Versiones de API: V1, V2

    Principal: salidas/marcas de agua

    Predeterminado: 1.0

    Valores válidos: 0.0 a 1.0

    Tipos de trabajo compatibles: Todos

    Ejemplo: 0.5

    Descripción:

    Haga que la marca de agua sea transparente estableciendo un valor de opacidad entre 0,0 (transparente) y 1.0 (opaco).

      {
    «input»: «s3: //zencodertesting/test.mov»,
    «productos»: [
      {
        «marcas de agua»: {
          «opacidad»: 0.5
        }
      }
    ]
  }

    Ver también: url
