filigranas

marcas de agua: Matriz o Hash

Versiones de API: V1, V2

Padre: salidas O opciones_de_perfil_dinámico

Valores válidos: Una matriz o hash de configuraciones de marca de agua

Tipos de trabajo compatibles: Todas

Descripción:

Puede agregar una o más marcas de agua a un video de salida utilizando nuestra API de marcas de agua. Zencoder admite hasta 4 archivos de marca de agua por salida en formato JPEG, BMP o PNG.

  • No se admiten GIF, imágenes de marca de agua RGB + alfa de 32 bits y RGB + alfa de 64 bits.
  • No se aplicarán marcas de agua que se extiendan más allá de los bordes del video. Se generará una advertencia en la salida si esto ocurre.
  • Al utilizar el códec VP6, existen algunas limitaciones adicionales. Solo puede aplicar una marca de agua al video (si se pasan varias marcas de agua, solo se usará la primera). Además, la escala de la marca de agua no es compatible y la imagen de la marca de agua debe ser un archivo PNG.
  { "input": "s3://zencodertesting/test.mov", "outputs": [ { "watermarks": [ { "url": "s3://bucket/watermark_file.png", "x": "20", "y": "-10%", "width": 32, "height": 24 }]}]}

url

url: String

Versiones de API: V1, V2

Padre: salidas / filigranas

Valores válidos: Una URL válida a un archivo de imagen.

Tipos de trabajo compatibles: Todas

Ejemplo:

  • https://s3.amazonaws.com/bucket/img.png
  • ftp: // usuario: pass@example.com/path/to/watermark.jpg

Descripción:

La URL de un archivo de imagen remoto para usar como marca de agua. Utilizar el aporte Sintaxis de URL. Admite S3, GCS, archivos en la nube, HTTP / S, FTP y SFTP, con o sin autenticación.

Zencoder admite archivos de marcas de agua en formato GIF, JPEG, BMP o PNG.

Para marcas de agua transparentes, use PNG con transparencia alfa o índice.

Formatos PNG compatibles

  • RGB de 8 bits
  • RGB + alfa de 8 bits
  • Escala de grises de 8 bits
  • Indexado de 8 bits
  • Indexado + Transparencia de 8 bits
  • RGB de 16 bits
  • Escala de grises de 16 bits

Formatos PNG no admitidos

  • Escala de grises de 8 bits + Alfa
  • Escala de grises de 16 bits + Alfa
  • RGB + Alfa de 16 bits
  { "input": "s3://zencodertesting/test.mov", "outputs": [ { "watermarks": { "url": "https://s3.amazonaws.com/bucket/img.png" }}]}

x

x: cadena o número

Versiones de API: V1, V2

Padre: salidas / filigranas

Defecto: -10

Tipos de trabajo compatibles: Todas

Ejemplo:

  • -0
  • 320
  • centrado

Descripción:

Dónde colocar la marca de agua dentro del video, en el eje x (izquierda / derecha). Puede ser un número de píxeles (por ejemplo, 100 o -20), un porcentaje del ancho del vídeo (por ejemplo, 25% o -5%) o «centrado' para alinear la marca de agua con el centro del vídeo. Utilice un número positivo para colocar en relación con el lado izquierdo del video y un número negativo para colocar en relación con el lado derecho del video. Utilice "-0" (como una cuerda) para bloquear en el lado derecho.

De forma predeterminada, la posición se basa en el área de contenido visible, sin incluir ningún relleno. Usar origen para establecer la posición de la marca de agua en función de la salida completa.

  { "input": "s3://zencodertesting/test.mov", "outputs": [ { "watermarks": { "x": 320 }}]}

ancho

ancho: Número o Cadena

Versiones de API: V1, V2

Padre: salidas / filigranas

Defecto: Escalar a altura o el ancho de la imagen original.

Valores válidos: Un número positivo o un porcentaje

Tipos de trabajo compatibles: Todas

Descripción:

El ancho de la marca de agua, expresado como un número de píxeles (por ejemplo, 64) o como un porcentaje del ancho del video (por ejemplo, 10%). Si altura se proporciona, pero no el ancho, la imagen de la marca de agua se escalará proporcionalmente.

  { "input": "s3://zencodertesting/test.mov", "outputs": [ { "watermarks": { "width": 100 }}]}

Ver también: altura

origen

origen: Cadena

Versiones de API: V1, V2

Padre: salidas / filigranas

Defecto: contenido

Valores válidos: contenido o marco

Tipos de trabajo compatibles: Todas

Ejemplo: marco

Descripción:

La parte del video en la que se basa el posicionamiento de la marca de agua. Esto solo afecta a los trabajos en los que aspect_mode es 'pad'.

  • 'frame' basa la colocación en la resolución completa de la salida, incluido cualquier acolchado.
  • «el contenido' basa la ubicación en el área de contenido visible, sin incluir el relleno.
  { "input": "s3://zencodertesting/test.mov", "outputs": [ { "watermarks": { "origin": "content" }}]}

opacidad

opacidad: flotar

Versiones de API: V1, V2

Padre: salidas / filigranas

Defecto: 1.0

Valores válidos: 0,0 a 1,0

Tipos de trabajo compatibles: Todas

Ejemplo: 0.5

Descripción:

Haga transparente la marca de agua estableciendo un valor de opacidad entre 0,0 (transparente) y 1,0 (opaco).

  { "input": "s3://zencodertesting/test.mov", "outputs": [ { "watermarks": { "opacity": 0.5 }}]}
