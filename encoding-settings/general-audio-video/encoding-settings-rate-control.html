    Configuración de control de velocidad

    Este tema contiene detalles sobre la configuración de codificación relacionada con el control de velocidad de codificación, que son importantes para la calidad de las salidas.

    quality

    quality:Integer

    Versiones de API: V1, V2

    Padres:outputs

    Predeterminado: 3

    Valores válidos: 1-5

    Tipos de trabajo compatibles: Todos

    Ejemplo: 4

    Descripción:

    La calidad de vídeo de salida deseada, de 1 a 5. Selecciona automáticamente un video_bitrate para lograr esta calidad.

    Esto elige automáticamente una velocidad de bits de vídeo que produce un vídeo de la calidad deseada. Esto es útil, ya que un vídeo puede necesitar una velocidad de bits más alta que otro para lograr una calidad visual objetivo, debido a la complejidad del contenido, la resolución, etc.

    Tenga en cuenta que si se establece y se establece, la configuración se ignorará.qualityvideo_bitratevideo_bitrate

    Una mayor calidad significa una tasa de bits más alta y viceversa.

    Calidad 1: Muy comprimida. Calidad visual mediocre, pero archivos pequeños.
    Calidad 2: Calidad aceptable.
    Calidad 3: Buena calidad. Mejor que la mayoría de los vídeos web.
    Calidad 4: Gran calidad. Se ve excelente.
    Calidad 5: Casi sin pérdidas. Archivos grandes. No se recomienda a menos que planee codificar esta salida de nuevo.

    Como regla general, reducir la calidad en un nivel reducirá el tamaño del archivo en aproximadamente un 40%. La codificación de mayor calidad también es un poco más lenta que la codificación de menor calidad.

    Recuerde que la velocidad de bits de salida variará de un archivo a otro cuando utilice la configuración de calidad, dependiendo del tipo de vídeo. Incluso en la misma configuración de calidad, el vídeo de baja complejidad (como un screencast) generalmente dará como resultado velocidades de bits más bajas que los vídeos de alta complejidad (como una película).

      {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "quality": 4
      }
    ]
  }

    Ver también:video_bitrate

    video_bitrate

    video_bitrate:Integer

    Versiones de API: V1, V2

    Padres:outputs

    Valores válidos: Un entero positivo. 100000 máx.

    Tipos de trabajo compatibles: Todos

    Ejemplo: 1200

    Descripción:

    La velocidad de bits de salida deseada para un vídeo, expresada en kbps. Esto da como resultado una velocidad de bits de salida predecible, pero no una calidad predecible. Por ejemplo, con 640x480, 500kbps podría ser suficiente para que un blog de vídeo se vea bien, pero una película de acción podría verse mal a la misma velocidad de bits. Del mismo modo, podría ser demasiado alto para un screencast, lo que resulta en un archivo que es más grande de lo necesario.

    Si una velocidad de bits confiable es más importante que una calidad confiable, elija video_bitrate. Si una calidad visual confiable es más importante, úselo quality en su lugar.

    Tenga en cuenta que si se establece y se establece, la configuración se ignorará.qualityvideo_bitratevideo_bitrate

    video_bitrate utiliza codificación de velocidad de bits variable (VBR) de dos pasos. La codificación one_pass se puede activar a través de la one_pass opción, aunque la codificación one_pass VBR es típicamente de mala calidad, y solo da como resultado una pequeña aceleración, por lo que esto no es recomendable.

    También tenga en cuenta que las velocidades de bits demasiado altas o demasiado bajas son inutilizables. De forma predeterminada, si especifica una tasa de bits demasiado baja, corregiremos automáticamente una tasa de bits aceptable. Habilite strict el modo si prefiere que un trabajo falle en esta situación en lugar de corregir automáticamente.

      {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "video_bitrate": 1200
      }
    ]
  }

    Ver también:quality

    audio_quality

    audio_quality:Integer

    Versiones de API: V1, V2

    Padres:outputs

    Predeterminado: 3

    Valores válidos: 1-5

    Tipos de trabajo compatibles: Todos

    Ejemplo: 4

    Descripción:

    La calidad de audio de salida deseada, de 1 a 5. Selecciona automáticamente un audio_bitrate para lograr esta calidad.

    Tenga en cuenta que si se establece y se establece, la configuración se ignorará.audio_qualityaudio_bitrateaudio_bitrate

    Quality 1 utiliza una velocidad de bits de audio baja, lo que da como resultado una calidad de sonido baja, pero archivos pequeños. La calidad 5 da como resultado una alta velocidad de bits de audio para archivos más grandes pero de mayor calidad.

    Recomendamos utilizar 3 para la mayoría de los propósitos, o 4-5 cuando se desea una calidad de sonido transparente.

      {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "audio_quality": 4
      }
    ]
  }

    Ver también: quality y audio_bitrate

    audio_bitrate

    audio_bitrate:Integer

    Versiones de API: V1, V2

    Padres:outputs

    Valores válidos: Un entero positivo.

    Tipos de trabajo compatibles: Todos

    Ejemplo: 96

    Descripción:

    Una configuración de velocidad de bits de salida, en Kbps. Con salida de 2 canales, esta es la velocidad de bits total de audio, no la velocidad de bits de cada canal, por lo que la salida estéreo de 128 kbps está codificada a 64 kbps por canal. Por debajo de 48 kbps por canal, la calidad puede comenzar a sufrir.

    Tenga en cuenta que si se establece y se establece, la configuración se ignorará.audio_qualityaudio_bitrateaudio_bitrate

    Tenga en cuenta que si configura manualmente un audio_sample_rate, debe elegir un audio_bitrate compatible, o la codificación puede fallar. De forma predeterminada, corregiremos automáticamente estos fallos y elegiremos una combinación de tasa de bits y frecuencia de muestreo válida. Habilite strict_mode para evitar esta corrección automática.

    Esta configuración no limita la velocidad de bits máxima del vídeo codificado. Si el sistema detecta que la velocidad de bits codificada final será mayor que este valor, reinicia internamente la codificación con esta velocidad de bits como la tasa de bits promedio de destino. Consulte decoder_bitrate_cap para limitar las velocidades de bits máximas.

    Para audio AAC, una tasa de bits inferior a 84 establecerá el perfil AAC en HE-AAC.

      {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "audio_bitrate": 96
      }
    ]
  }

    Consulte también: audio_quality , audio_sample_rate , y max_aac_profile

    max_video_bitrate

    max_video_bitrate:Integer

    Versiones de API: V1, V2

    Padres:outputs

    Valores válidos: Un entero positivo. 100000 máx.

    Tipos de trabajo compatibles: Todos

    Ejemplo: 1500

    Descripción:

    Velocidad de bits media máxima para una película. Sobrescribe la configuración de calidad y video_bitrate para asegurarse de que una velocidad de bits no supera el número proporcionado.

    max_video_bitrate es especialmente útil cuando se codifica para dispositivos móviles mediante la quality configuración, que selecciona automáticamente una velocidad de bits. Los dispositivos móviles a veces tienen límites fijos de velocidad de bits; por ejemplo, el iPhone 3GS tiene un límite de velocidad de bits de 1500 kbps.

    Nota: Esta opción no es compatible con códecs VP6 o HEVC.

      {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "max_video_bitrate": 1500
      }
    ]
  }

    speed

    speed:Integer

    Versiones de API: V1, V2

    Padres:outputs

    Predeterminado: 3

    Valores válidos: 1-5

    Tipos de trabajo compatibles: vod

    Ejemplo: 4

    Descripción:

    Una velocidad de transcodificación objetivo. La transcodificación más lenta permite una compresión de archivos más avanzada, mientras que la transcodificación más rápida es posible omitiendo algunas funciones de compresión avanzadas. Los valores válidos son 1-5.

    Nota: por el momento, sólo la salida H.264 tiene cinco niveles de velocidad. Para el contenido VP6, 1-2 son un modo más lento y 3-5 son un modo más rápido. Otros códecs de vídeo no admiten esta configuración en absoluto.

      {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "speed": 4
      }
    ]
  }

    decoder_bitrate_cap

    decoder_bitrate_cap:Integer

    Versiones de API: V1, V2

    Padres:outputs

    Valores válidos: Un entero positivo. 100000 máx.

    Tipos de trabajo compatibles: Todos

    Ejemplo: 800

    Descripción:

    La velocidad de bits máxima que se alimenta al decodificador a través de un búfer. Esta configuración se utiliza normalmente sólo para la transmisión (RTMP, HLS o vídeo de difusión).

    Utilice esta configuración únicamente si comprende sus implicaciones, ya que puede disminuir la calidad del vídeo.

    Para las salidas en vivo, este valor predeterminado es un 15% mayor que el video_bitrate, si se especifica video_bitrate.

      {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "decoder_bitrate_cap": 800
      }
    ]
  }

    Consulte también: decoder_buffer_size

    decoder_buffer_size

    decoder_buffer_size:Integer

    Versiones de API: V1, V2

    Padres:outputs

    Valores válidos: Un entero positivo. 100000 máx.

    Tipos de trabajo compatibles: Todos

    Ejemplo: 800

    Descripción:

    El tamaño del búfer alimentado al decodificador cuando se utiliza un bitrate_cap, expresado en kbps. El tamaño_buffer_dividido por bitrate_cap representa el tamaño del búfer en segundos; por lo tanto, si establece bitrate_cap en 1000 y buffer_size en 1000, el búfer es efectivamente 1,0 segundo. Si bitrate_cap es 500 y buffer_size es 1000, el búfer es de 2.0 segundos.

    Utilice esta configuración únicamente si comprende sus implicaciones, ya que puede disminuir la calidad del vídeo. Normalmente, esto solo se debe usar para la transmisión (o para la reproducción de dispositivos).

    Para las salidas en vivo, este valor predeterminado es 300% del decoder_bitrate_cap, creando una duración de búfer de 3 segundos. Sin embargo, si segment_seconds también se establece, el tamaño decoder_buffer_de_predeterminado se reducirá de manera que la duración efectiva del búfer sea la mitad de la duración de un segmento.

      {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "decoder_buffer_size": 800
      }
    ]
  }

    Ver también: decoder_bitrate_cap

    one_pass

    one_pass:Boolean

    Versiones de API: V1, V2

    Padres:outputs

    Predeterminado:false

    Valores válidos: otruefalse

    Tipos de trabajo compatibles: vod

    Ejemplo:true

    Descripción:

    Por defecto, usaremos la codificación de dos pasadas cada vez que codifiquemos en un destino video_bitrate, y la codificación one_pass al realizar la quality codificación constante (que no se beneficia de una segunda pasada). Esta opción forzará la codificación one_pass al apuntar a un video_bitrate específico.

    Recomendamos encarecidamente no forzar la codificación one_pass. La primera pasada en la codificación de dos pasos es más rápida que la segunda pasada, por lo que pasar de la codificación de dos pasos a la codificación one_pass solo da como resultado una aceleración de codificación del 25%, no un 50% más rápida en la codificación. Y la codificación de dos pasos se ve significativamente mejor que la codificación one_pass.

      {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "one_pass": true
      }
    ]
  }

    audio_constant_bitrate

    audio_constant_bitrate:Boolean

    Versiones de API: V1, V2

    Padres:outputs

    Predeterminado:false

    Valores válidos: otruefalse

    Tipos de trabajo compatibles: Todos

    Ejemplo:true

    Descripción:

    Habilite el modo de velocidad de bits constante (CBR) para audio, cuando sea posible. audio_sample_rate puede ajustarse para tener en cuenta la compatibilidad.

    audio_bitrate también debe especificarse al utilizar esta opción.

    Compatible con códecs de audioAAC y MP3. No es compatible cuando se utiliza con el VP6 video_codec.) Para audio MP3, solo ciertas velocidades de bits son compatibles con esta opción: 8, 16, 24, 32, 40, 48, 56, 64, 80, 96, 112, 128, 160, 192, 224, 256, 320.

    Nota: El códec AAC no define un modo CBR fijo, por lo que todavía hay pequeñas variaciones en la velocidad de bits durante periodos de tiempo cortos. La mayoría del software aún informará de estas codificaciones como VBR.

      {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "audio_constant_bitrate": true
      }
    ]
  }
