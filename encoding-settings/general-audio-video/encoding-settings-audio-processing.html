audio_gain

audio_gain:Float

Versiones de API: V1, V2

Padre: salidas

Valores válidos: Un número entre -60 y 60

Tipos de trabajo compatibles: Todas

Ejemplo: 5

Descripción:

Aplica una cantidad de ganancia al audio. Esto aumenta o disminuye el volumen del audio en la cantidad especificada después de aplicar otros efectos. Esto es principalmente útil para ajustar los niveles después de aplicar otros efectos o realizar ajustes en archivos para los que ya conoce el nivel. Los sonidos que superen un poco el volumen máximo se limitarán suavemente para preservar la fidelidad, pero si se aumentan significativamente por encima del volumen máximo, se producirá un sonido distorsionado. Ver audio_normalize para aumentar el volumen automáticamente sin distorsión. Especificado en dB, positivo o negativo, hasta 60 dB.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_gain": 5
    }
  ]
}

Ver también: audio_normalize

audio_normalize

audio_normalize:Boolean

Versiones de API: V1, V2

Padre: salidas

Defecto: falso

Valores válidos: verdadero o falso

Tipos de trabajo compatibles: vod

Ejemplo: cierto

Descripción:

Normaliza el audio a 0dB. Esto aumenta el volumen del audio tanto como sea posible sin causar distorsión. Para ayudar a asegurar un ajuste consistente mediante los efectos de expansión y compresión, normalizamos el volumen antes y después de esos efectos. Ver audio_pre_normalize y audio_post_normalize para normalizar solo en uno de esos puntos. audio_normalize es equivalente a especificar tanto audio_pre_normalize como audio_post_normalize.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_normalize": true
    }
  ]
}

Consulte también: audio_gain, audio_pre_normalize y audio_post_normalize

audio_highpass

audio_highpass:Integer

Versiones de API: V1, V2

Padre: salidas

Valores válidos: Una frecuencia en Hz, de 5 a 24000

Tipos de trabajo compatibles: Todas

Ejemplo: 10

Descripción:

Aplique un filtro de paso alto al audio a la frecuencia especificada en Hz. Esto evitará que se incluyan en la salida frecuencias de audio por debajo del umbral especificado. Un uso muy común de esta configuración es para reducir el retumbar / ruido del audio, como los sonidos de camiones conduciendo afuera durante la grabación o algunos sonidos causados por el viento o golpear un micrófono. DC-Shift también se puede reducir / eliminar especificando un filtro de paso alto a una frecuencia baja, como 10 Hz.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_highpass": 10
    }
  ]
}

Ver también: audio_lowpass

audio_lowpass

audio_lowpass:Integer

Padre: salidas

      "audio_lowpass": 3000

audio_compression_ratio

audio_compression_ratio:Float

Versiones de API: V1, V2

Padre: salidas

Valores válidos: 1,0 a 30.0

Tipos de trabajo compatibles: Todas

Ejemplo: 2.5

Descripción:

Comprima el rango dinámico del audio reduciendo el volumen a una proporción especificada de N: 1, por encima de la especificada umbral_de_compresión_de_audio. Las proporciones entre 1,0 y 30,0 son válidas. La compresión es esencialmente útil para hacer que el volumen de audio sea más consistente entre los puntos silenciosos y los puntos ruidosos. Esto se puede usar para hacer que los lugares ruidosos (gritos, portazos, etc.) sean menos discordantes. Usado junto con audio_gain o audio_normalization, puede hacer que las partes silenciosas de un discurso sean más audibles.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_compression_ratio": 2.5
    }
  ]
}

Ver también: umbral_de_compresión_de_audio

audio_compression_threshold:Float

audio_karaoke_mode

audio_karaoke_mode:Boolean

Versiones de API: V1, V2

Padre: salidas

Defecto: falso

Valores válidos: verdadero o falso

Tipos de trabajo compatibles: Todas

Ejemplo: cierto

Descripción:

Aplique un efecto de "karaoke" al audio restando el canal de audio derecho del izquierdo y manteniendo solo la diferencia. Solo se aplica a salidas estéreo.

Tenga cuidado al usar esta función:

  • A veces produce distorsión
  • A veces no hace nada en absoluto
  • A veces afecta negativamente el ruido de fondo o la música.
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_karaoke_mode": true
    }
  ]
}
