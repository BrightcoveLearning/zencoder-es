    Configuración de transmisión segmentada

    Este tema contiene detalles sobre la configuración de codificación relacionada con la transmisión segmentada.

    Resumen

    Zencoder soporta varias formas de transmisión adaptativa y segmentada.

    • Streaming y listas de reproducción HTTP Live (segmentadas)
    • Transmisión fluida de Microsoft
    • Vientos Altos SMIL

    Cada formato de transmisión utiliza un subconjunto diferente de las opciones de API que se enumeran a continuación. Consulte las guías de cada tipo de transmisión para obtener más información.

    Nota: algunos reproductores no manejan el caso en el que las longitudes de transmisión de audio y vídeo difieren en más de la duración de un segmento. Si encuentra esto, intente usar una longitud de segmento más corta.

    NOTAS

    • Un trabajo debe tener al menos una salida estándar o segmentada; la creación de trabajos con sólo salidas de lista de reproducción no funcionará.
    • Las salidas de la lista de reproducción ignorarán la mayoría de las opciones de salida, excepto url, base_url, filename, streams, public, y type.
    • Si se carga una salida segmentada al almacenamiento temporal, se cargará como un archivo ZIP.

    streaming_delivery_format

    streaming_delivery_format:String

    Versiones de API: V1, V2

    Padres:outputs

    Valores válidos: guión

    Tipos de trabajo compatibles: Todos

    Ejemplo: guión

    Descripción:

    Establece el formato/protocolo para una salida que se entregará mediante una configuración de transmisión específica, incluidos los manifiestos necesarios, estructuras de directorios u otros paquetes.

    Para las salidas 'dash', esto también cambiará el segment_seconds predeterminado a 2, aplicará intervalos de fotogramas clave fijos, incluirá un manifiesto MPD y creará una estructura de directorios con archivos de inicialización y segmentos.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "streaming_delivery_format": "dash"
    }
  ]
}

    Consulte también: streaming_delivery_profile

    streaming_delivery_profile

    streaming_delivery_profile:String

    Versiones de API: V1, V2

    Padres:outputs

    Predeterminado: vivo

    Valores válidos: live, hbbtv_1.5, on_demand

    Tipos de trabajo compatibles: Todos

    Ejemplo: Live

    Descripción:

    Establece el perfil del formato de entrega de streaming, asegurando que se seleccionan las opciones de compatibilidad con el perfil. Las opciones válidas dependen del seleccionado streaming_delivery_format.

    Para las dash salidas, los perfiles admitidos actualmente son:

    • live - Establecerá el DASH perfil en «urn:mpeg:dash:perfile:isoff-live:2011" y usará un solo SegmentTemplate para cada uno AdaptationSet. Para DRM, utilizará un IV de 8 bytes, y para PlayReady usará la versión 1.1, incluyendo un átomo PSSH en los medios.
    • hbbtv_1.5 - Establecerá el DASH perfil en «urn:hbbtv:dash:profile:isoff-live:2012" e incluirá los atributos necesarios para las representaciones de audio y video. Para DRM, utilizará un IV de 16 bytes, y para PlayReady utilizará la versión 1.0, excluyendo los átomos de PSSH.
    • on_demand - Establecerá el DASH perfil en «urn:mpeg:dash:profile:isoff-on-demand:2011" y codificará los medios como archivos fragmentados individuales para cada secuencia en lugar de segmentarlos. De lo contrario, lo mismo que el perfil en vivo.
    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "streaming_delivery_format": "dash",
      "streaming_delivery_profile": "hbbtv_1.5"
    }
  ]
}

    Ver también:streaming_delivery_format

    segment_seconds

    segment_seconds:Float

    Versiones de API: V1, V2

    Padres:outputs

    Predeterminado: 10

    Valores válidos: Números entre 1,0 y 3600.0; los valores inferiores a 2.0 sólo se permiten cuando byte_range_segmenting está habilitado. Debido a que se ha encontrado que segmentos muy cortos causan problemas de reproducción, el segmento final puede ser más largo que el valor especificado.

    Tipos de trabajo compatibles: Todos

    Ejemplo: 8

    Descripción:

    Establece la duración máxima de cada segmento en una salida segmentada.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "segment_seconds": 8
    }
  ]
}

    alternate_audio

    alternate_audio:Hash

    Versiones de API: V1, V2

    Padres:outputs

    Valores válidos: Hash de grupos de audio alternativos, cada uno de los cuales es una matriz de hashes de reproducción de secuencias de audio

    Tipos de trabajo compatibles: Todos

    Descripción:

    Proporciona un conjunto de secuencias de audio alternativas para HLS listas de reproducción. max_hls_protocol_version Requiere ser 4 o más.

    Esta opción insertará EXT-X-MEDIA etiquetas en la lista de reproducción. Si no name se establece, entonces se generará un nombre basado en el dado language.

    Opciones válidas para copias de audio alternativas

    • language: Código ISO 639 para el idioma de la pista de audio, como «en», «es» o «zh».
    • name ( opcional): El nombre del grupo de audio alternativo. Si no se establece nombre, este será el nombre de idioma predeterminado.
    • path ( opcional): Ruta opcional a la lista de reproducción M3U8 para esta representación de audio. Si la ruta no está configurada, esta secuencia de audio alternativa describe el audio contenido en la secuencia de origen.
    • source ( opcional): Etiqueta en una salida de medios del mismo trabajo. Esto se puede usar para configurar el códec para las copias que utilizan esta secuencia de audio.
    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "playlist",
      "alternate_audio": {
        "custom_audio_group": [
          {
            "source": "english-audio",
            "path": "alternate-audio-en.m3u8",
            "language": "en"
          },
          {
            "source": "french-audio",
            "path": "alternate-audio-fr.m3u8",
            "language": "fr"
          }
        ],
        "secondary_audio_group": {
          "path": "path/to/secondary-audio-group.m3u8",
          "language": "en",
          "name": "Secondary English Audio Track",
          "source": "secondary-english-audio"
        }
      }
    }
  ]
}

    max_hls_protocol_version

    max_hls_protocol_version:Integer

    Versiones de API: V1, V2

    Principal:/flujosoutputs

    Predeterminado: 5 para el cifrado AES de muestra; de lo contrario, 2.

    Valores válidos: 2-5

    Tipos de trabajo compatibles: Todos

    Ejemplo: 4

    Descripción:

    Establece la versión máxima HLS del protocolo que se va a utilizar. Las características especiales se utilizarán según estén disponibles.

    Esta opción debe ser mayor o igual a hls_protocol_version cuando se proporciona.

    Nota: Sólo se aplica a las salidas HTTP Live Streaming.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "path": "file-240k.m3u8",
      "bandwidth": 240,
      "max_hls_protocol_version": 4
    },
    {
      "type": "segmented",
      "path": "file-64k.m3u8",
      "bandwidth": 64,
      "max_hls_protocol_version": 4
    },
    {
         "filename":"playlist.m3u8",
         "public":1,
         "max_hls_protocol_version":4,
         "streams":[
            {
               "path":"file-240k.m3u8"
            },
            {
               "path":"file-64k.m3u8"
            }
         ],
         "type":"playlist"
    }
  ]
}

    Ver también:hls_protocol_version

    hls_protocol_version

    hls_protocol_version:Integer

    Versiones de API: V1, V2

    Principal:/flujosoutputs

    Predeterminado: Automático según la max_hls_protocol_version configuración

    Valores válidos: 2-5

    Nota: HLS las versiones 6 y 7 no son totalmente compatibles; si las especifica para hls_protocol_version el único efecto será eliminar las entidades obsoleta de la lista de reproducción.

    Tipos de trabajo compatibles: Todos

    Ejemplo: 3

    Descripción:

    Establece la versión del protocolo HLS que se va a utilizar. Las características especiales se utilizarán según estén disponibles.

    Nota: Las versiones 1-3 de HLS utilizan un entero para duration en las listas de reproducción HLS. Las versiones 4 y posteriores usan un número de coma flotante.

    Nota: Sólo se aplica a las salidas HTTP Live Streaming.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "hls_protocol_version": 3
    }
  ]
}

    Ver también:max_hls_protocol_version

    prepare_for_segmenting

    prepare_for_segmenting:String or Array

    Versiones de API: V1, V2

    Padres:outputs

    Valores válidos: hls , mss , dash

    Tipos de trabajo compatibles: Todos

    Ejemplo:hls,dash

    Descripción:

    Si planea transmux una salida a HLS MSS y/o DASH después de codificarla con subtítulos en MP4, establezca esta opción en true para incluir subtítulos y temporización de fotogramas clave para segmentar. Esto permite que las versiones segmentadas incluyan subtítulos compatibles con HLS a pesar de la transmuxación, y proporciona la segmentación a intervalos de tiempo consistentes.

    Especifique el tipo de salida (s) segmentada (s) que esta salida se transmuxará posteriormente para crear, de modo que Zencoder pueda preparar los subtítulos y fotogramas clave según sea necesario.

    Nota: Actualmente no se pueden agregar subtítulos a las HLS salidas al transmuxing.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "prepare_for_segmenting": "hls,dash"
    }
  ]
}

    Consulte también: caption_url y skip_captions

    smil-base-url

    smil_base_url:String

    Versiones de API: V1, V2

    Padres:outputs

    Tipos de trabajo compatibles: Todos

    Ejemplo: rtmp: //127.0.0.1/base/ruta/

    Descripción:

    Esto agregará la etiqueta < meta base="smil_base_url_value»/> a la > sección < principal de una lista de reproducción SMIL.

    Este ejemplo mostrará una lista de reproducción que se ve así:

      <smil >
    <cabeza >
      <meta base="rtmp: //123.456.123.123/ruta/a/vídeos/»/>
    </cabeza >
    <cuerpo >
      <conmutador >
        <video src="mp4:test.mp4" system-bitrate="500000"/ >
      </conmutador >
    </cuerpo >
  </smil >

    Nota: sólo funciona en una salida de tipo playlist, con un formato de vientos fuertes.

    generate_keyframe_manifest

    generate_keyframe_manifest:Boolean

    Versiones de API: V2

    Padres:outputs

    Predeterminado:false

    Valores válidos: otruefalse

    Tipos de trabajo compatibles: Todos

    Ejemplo:true

    Descripción:

    Genera un manifiesto de HLS fotograma clave (fotograma I) que es necesario para la reproducción hacia delante y hacia atrás.

    El nombre de archivo de manifiesto de fotogramas clave predeterminado es el mismo que el nombre de archivo de la lista de reproducción, pero con '-iframe-index' añadido (por lo que para una lista de reproducción 'master.m3u8', el manifiesto de fotogramas clave sería 'master-iframe-index.m3u8'). El nombre de archivo del manifiesto de fotogramas clave se puede anular con keyframe_manifest_filename.

    Habilitado de forma predeterminada si max_hls_protocol_version o hls_protocol_version está establecido 4 o superior.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "format": "ts",
      "generate_keyframe_manifest": true,
      "keyframe_manifest_filename": "my_iframe_index.m3u8"
    }
  ]
}

    Ver también:keyframe_manifest_filename

    keyframe_manifest_filename

    keyframe_manifest_filename:String

    Versiones de API: V2

    Padres:outputs

    Predeterminado:iframe_index.m3u8

    Tipos de trabajo compatibles: Todos

    Ejemplo:my_iframe_index.m3u8

    Descripción:

    A menos que se especifique, el nombre de archivo utilizado para el manifiesto de fotogramas clave es el mismo que el nombre de archivo de la lista de reproducción, pero con '-iframe-index' anexado. Por ejemplo, para la lista de reproducción 'master.m3u8', el manifiesto de fotogramas clave sería 'master-iframe-index.m3u8'.

    Esta opción requiere que generate_keyframe_manifest sea true, o hls_protocol_version sea 4 o superior.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "format": "ts",
      "generate_keyframe_manifest": true,
      "keyframe_manifest_filename": "my_iframe_index.m3u8"
    }
  ]
}

    Vea también: generate_keyframe_manifest

    allow_skipped_sources

    allow_skipped_sources:Boolean

    Versiones de API: V2

    Padres:outputs

    Predeterminado:false

    Valores válidos: otruefalse

    Tipos de trabajo compatibles: Todos

    Ejemplo:true

    Descripción:

    Al generar la lista de reproducción, en lugar de fallar la salida si alguna de las secuencias a las que se hace referencia se ha omitido (como parte de la funcionalidad de salidas condicionales ), simplemente déjelas fuera de la lista de reproducción.

    Para ignorar correctamente las fuentes omitidas, las secuencias de la lista de reproducción deben tener un source definido.

    allow_skipped_sources no es compatible con la salida SMIL. SMIL no reconocerá las fuentes omitidas.
    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "playlist",
      "allow_skipped_sources": true,
      "streams": [
        {
          "path": "low/index.m3u8",
          "source": "low-hls-rendition"
        },
        {
          "path": "high/index.m3u8",
          "source": "high-hls-rendition"
        }
      ]
    }
  ]
}

    Ver también:source
