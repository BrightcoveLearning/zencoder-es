Descripción general

Transmuxing es el proceso de cambiar el formato de un archivo de video o audio mientras se preservan algunas o todas las transmisiones del archivo original.

En la actualidad, Zencoder puede transmudar pistas de audio de vídeo H.264 y AAC/MP3 de formatos MP4/MOV a MP4/TS (incluida la creación de una salida MP4 fragmentada o TS segmentada). Sin embargo, hay una serie de requisitos que deben cumplir los archivos de entrada para transmuxar correctamente:

Video

  • Formato contenedor MP4/MOV
  • Códec H.264
  • No se utilizan marcos b

Audio

  • Formato contenedor MP4/MOV
  • Códec AAC o MP3

Segmentar / Fragmentar

  • El GOP máximo debe ser menor o igual que la duración del segmento
  • No se puede utilizar HE-AAC-V2

Otro

  • La mayoría de las opciones de codificación no tienen ningún efecto en los archivos transmuxed porque las transmisiones no se vuelven a codificar

copy_video

copy_video:Boolean

Versiones de API: V1, V2

Padre: producción

Defecto: falso

Valores válidos: verdadero o falso

Tipos de trabajo compatibles: Todas

Ejemplo: cierto

Descripción:

Especificando copy_video tomará la pista de video del archivo de video de entrada y la transmitirá al archivo de salida resultante.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "copy_video": true
    }
  ]
}
