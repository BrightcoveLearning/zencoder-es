    Configuración general de salida

    Este tema contiene detalles sobre la configuración de codificación relacionada con la configuración de salida general (que se aplica a todas las salidas).

    type

    type:String

    Versiones de API: V1, V2

    Padres:outputs

    Valor predeterminado: estándar

    Valores válidos: standard , segmented , captions , playlist y transfer-only

    Tipos de trabajo compatibles: Todos

    Ejemplo:segmented

    Descripción:

    Por defecto, Zencoder considerará que las salidas son archivos de audio o vídeo estándar. Actualmente, hay dos tipos especiales de productos: segmented y playlist.

    • segmented: El caso de uso típico de la segmentación es para la transmisión HTTP en vivo de Apple (consulte Transmisión segmentada). Para salidas segmentadas, el formato predeterminado se cambiará a ts en lugar de mp4.
    • captions: Sólo API V2
      El tipo de salida de subtítulos crea un archivo que contiene sólo los subtítulos del vídeo de origen, en lugar de un vídeo con subtítulos añadidos. Nota: las salidas de tipos de subtítulos pueden no ser las únicas salidas para un trabajo.
      Los subtítulos se pueden proporcionar a Zencoder como un archivo independiente (utilizando la opción caption_url ), o se extraerán del vídeo fuente de forma predeterminada.
      Los formatos admitidos para extraer subtítulos del vídeo de origen son MP4 y TS. En un archivo MP4, los subtítulos pueden estar ubicados en una pista independiente (en formato CEA-608); de lo contrario, los subtítulos de un archivo MP4 o TS deben estar incrustados en una secuencia de vídeo MPEG2 o AVC, conforme a los estándares A/53 o SCTE-128 (CC1 y CC3).
      Formatos de salida admitidos para salidas de tipo subtítulos: 'mp4','scc','webvtt','vtt','m3u8','dfxp'.
    • lista de reproducción: El tipo de salida de la lista de reproducción se utiliza en flujos de salida de velocidad de bits múltiple para generar el archivo de lista de reproducción M3U8 maestro. Nota: las salidas de tipo lista de reproducción pueden no ser las únicas salidas para un trabajo.
    • transfer-only: API V2 Sólo
      una transfer-only salida omite la codificación y envía el vídeo original a un archivo url.
      Una transfer-only salida también se puede utilizar source para transferir el vídeo de otra salida a la especificada url , en lugar de utilizar la entrada.
    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented"
    }
  ]
}

    label:String

    Versiones de API: V1, V2

    Padres:outputs

    Valores válidos: Una cadena

    dynamic_profile_rendition

    dynamic_profile_rendition:Integer

    Versiones de API: V2

    Padres:outputs

    Tipos de trabajo compatibles: Todos

    Descripción:

    La dynamic_profile_rendition configuración especifica que esta salida debe utilizar la configuración del perfil CAE generado dinámicamente, y específicamente la representación del perfil desde la que se utilizará la configuración. Cuando dynamic_profile_rendition está configurado, no es necesario especificar configuraciones de codificación como resolution , bitrate etc. (Consulte la Guía CAE para ver la lista completa de ajustes que establece automáticamente CAE).

    {
  "input": "s3://zencodertesting/test.mov",
  "generate_dynamic_profile": true,
  "dynamic_profile_options":{ "max_renditions": 6 },
  "outputs" :[
  {"dynamic_profile_rendition": 1},
  {"dynamic_profile_rendition": 2},
  {"dynamic_profile_rendition": 3},
  {"dynamic_profile_rendition": 4},
  {"dynamic_profile_rendition": 5},
  {"dynamic_profile_rendition": 6}
  ]
}

    url

    url:String

    Versiones de API: V1, V2

    Padres:outputs

    Valores válidos: Una dirección válida de S3, Cloud Files, GCS, FTP, FTP, SFTP, Aspera, Azure o HTTP.

    Tipos de trabajo compatibles: Todos

    Ejemplo:

    • s3://my-output-bucket/final.mp4
    • cf://username:api_key@container/object
    • cf+uk://username:api_key@container/object
    • gcs://access_key:secret_key@gcs-bucket/object
    • ftp://user:password@ftp.example.com/final.mp4
    • sftp://user:password@sftp.example.com/final.mp4
    • aspera://user:password@host:ssh-port/path/to/file.mp4

    Descripción:

    Una URL completa, incluido un nombre de archivo, a la que se cargará el archivo de salida. Consulte base_url para obtener más información sobre los proveedores.

    Apoyamos oficialmente la carga HTTP a S3, GCS, Cloud Files y Akamai.

    Notas sobre S3:

    Debe conceder permisos de escritura a Zencoder en el depósito de salida antes de enviar un trabajo a Zencoder. Tenga en cuenta que es el depósito de S3, y no la «ruta» de S3 lo que necesita permiso de escritura. Por ejemplo, un activo en http://s3.amazonaws.com/my_files/0000/video.mp4 necesita acceso de escritura agregado a «my_files» (el nombre del bucket), no «mi_archivos/0000" (la ruta).

    Para crear claves que contengan caracteres especiales que no son válidos en URL (? o #), debe codificarlos por URL (%3F y%23, respectivamente). Todos los demás caracteres no deben estar codificados por URL.

    Puede proporcionar la región del bucket de S3 estableciendo el protocolo de la URL en, donde este documento define «region-name».s3+region-name Por ejemplo, para acceder a un archivo desde un depósito en la región Asia Pacífico (Singapur), establezca la dirección URL en s3+ap-southeast-1://my-bucket/file.mp4.

    Notas sobre archivos de nube:

    Puede especificar la región que se va a utilizar (DFW, ORD o Reino Unido) agregándola al protocolo, como.cf+ord://username:api_key@container/object Actualmente, las únicas regiones de Cloud Files admitidas son DFW, ORD y Reino Unido. La región DFW se usará de forma predeterminada.

    Cualquier carácter especial en el nombre del contenedor debe estar codificado por URL.

    Notas sobre el almacenamiento en la nube de Google:

    Las claves de acceso GCS se pueden encontrar en la sección «Acceso interoperable» de la consola de API de Google. Las claves de acceso deben ser escapadas correctamente.

    Los nombres de depósito de GCS que incluyen puntos no son compatibles.

    Notas sobre Azure:

    Si la URL contiene autenticación con caracteres especiales, asegúrese de que se escapan correctamente.

    Notas sobre FTP:

    Si la URL contiene autenticación con caracteres especiales, asegúrese de que se escapan correctamente.

    Tratamos de escribir desde la raíz de su servidor, así que use una ruta absoluta para su URL para asegurarse de que podemos escribir en su servidor con éxito.

    Notas sobre Aspera:

    La porción ssh-port de la URL no es necesaria, y tendrá por defecto 22.

    Zencoder ejecuta un Aspera Client en los servidores de codificación de vídeo, que se conecta a su Aspera Enterprise o Connect Server, que aloja los archivos.

    Esta es una situación similar a la que ejecuta un servidor FTP para hacer que sus archivos estén disponibles y Zencoder usando un cliente FTP para la transferencia de archivos.

    Notas sobre Akamai NetStorage a través de HTTPS:

    Zencoder puede cargar contenido a Akamai NetStorage a través de FTP o HTTPS. Para cargar contenido a través de HTTPS, construya una URL de salida como https://[Host]-nsu.akamaihd.net/[CPCode]/[path]/[file.ext].

    Puede hacer referencia a su nombre de clave y clave de Akamai NetStorage como Servidor externo cartas credenciales configurando el Usuario de servidor externo en el campo Akamai NetStorage Keyname y el Contraseña del servidor externo a la clave de Akamai NetStorage.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "url": "s3://my-output-bucket/final.mp4"
    }
  ]
}

    Ver también:secondary_url

    base_url

    base_url:String

    Versiones de API: V1, V2

    Padres:outputs

    Valores válidos: Una URL válida de S3, Cloud Files, GCS, Aspera, FTP, FTPS o SFTP, sin nombre de archivo.

    Tipos de trabajo compatibles: Todos

    Ejemplo:

    • s3://my-output-bucket/
    • cf://username:api_key@container/
    • gcs://access_key:secret_key@gcs-bucket/
    • ftp://user:password@ftp.example.com/path/to
    • http://s3.amazonaws.com/my-output-bucket/
    • aspera://user:password@host:ssh-port/path/to/

    Descripción:

    Una dirección URL de directorio base S3, Cloud Files, GCS, FTP, FTPS o SFTP donde se cargará el archivo transcodificado, sin nombre de archivo. Si no se especifica nombre de archivo , se generará un nombre de archivo aleatorio más una extensión apropiada.

    package_filename

    package_filename:String

    Versiones de API: V1, V2

    Padres:outputs

    Valores válidos: Un nombre de archivo de paquete válido.

    Tipos de trabajo compatibles: Todos

    Ejemplo: final.zip

    Descripción:

    El nombre de archivo de una salida empaquetada. Si la extensión coincide con un valor válido package_format, se establecerá el valor predeterminado para package_format también. Cuando se especifica, este valor se utiliza en combinación con base_url para generar una dirección URL de salida, en lugar de filename. Si no filename se especifica, el valor predeterminado será el mismo que package_filename , pero con la extensión de archivo correcta para el tipo de medio.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "package_filename": "final.zip"
    }
  ]
}

    Ver también: base_url y nombre del archivo

    package_format

    package_format:String

    Versiones de API: V1, V2

    Padres:outputs

    Valores válidos: zip y tar

    Tipos de trabajo compatibles: Todos

    Ejemplo:zip

    Descripción:

    Crea un zip o tar que contiene todos los archivos multimedia de una salida y carga este paquete único en lugar de todos los archivos individuales. Las miniaturas sin separadas se base_url incluirán en subdirectorios dentro del archivo del paquete también.

    Si package_filename no se especifica, el nombre de archivo del paquete se generará en función del nombre de archivo (pero con el formato del paquete como extensión). Consulte package_filename para obtener más información sobre el manejo de URL.

    Nota: Cuando se incluyen miniaturas en el archivo del paquete, las rutas dentro del paquete se devolverán a través de la API como su URL, en lugar de una URL completa.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "package_format": "zip"
    }
  ]
}

    Ver también:package_filename

    strict

    strict:Boolean

    Versiones de API: V1, V2

    Padres:outputs

    Predeterminado:false

    Valores válidos: otruefalse

    Tipos de trabajo compatibles: Todos

    Ejemplo:true

    Descripción:

    Habilite el modostrict" ". De forma predeterminada, intentaremos corregir los problemas de codificación por usted. Esto se llama modocompatible" ". Por ejemplo, si establece una velocidad de bits de vídeo demasiado baja para nuestro codificador, corregiremos automáticamente y usaremos la velocidad de bits más baja posible. Por lo tanto, si pide una velocidad de bits de 150 kbps, pero el codificador no puede hacer nada inferior a 200 kbps, codificaremos automáticamente el video a 200kbps para usted.

    Si habilita el modostrict" ", no corregiremos la configuración de codificación como esta. En cambio, fallaremos el trabajo, porque no podemos cumplir con su especificación objetivo.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "strict": true
    }
  ]
}

  "outputs": [
    {
      "skip_video": true
    }
  ]
}

    source

    source:String

    Versiones de API: V2

    Padres:outputs

    Tipos de trabajo compatibles: Todos

    Ejemplo:parent

    Descripción:

    Hace referencia a una etiqueta en otra salida y utiliza el vídeo creado por esa salida para procesarlo en lugar del archivo de entrada.

    El uso de la fuente junto con un type de «solo transferencia» permite transferir un vídeo de salida única a un segundo url. Útil para enviar un solo vídeo a varios destinos.

    {
  "input": "s3://zencodertesting/test.mov",
  "output": [
    {
      "label": "parent",
      "url": "s3://bucket/video.mp4"
    },
    {
      "source": "parent",
      "url": "s3://other-bucket/video.mp4",
      "type": "transfer-only"
    }
  ]
}

    Consulte también: etiqueta y type

    credentials

    credentials:String

    Versiones de API: V2

    Padres:outputs

    Tipos de trabajo compatibles: Todos

    Ejemplo:s3_production

    Descripción:

    Hace referencia al apodo de utilizar saved credentials para la transferencia.

    Las credenciales solo se pueden usar con URL que coincidan con su tipo; por ejemplo, las credenciales de S3 emparejadas con una URL FTP serán rechazadas.

    Nota: Las credenciales más «precisas» especificadas tienen prioridad. Por ejemplo, las credenciales pasadas en la URL tendrán prioridad sobre el parámetro credenciales, del mismo modo que el parámetro credenciales tiene prioridad sobre las credenciales predeterminadas.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "url": "s3://bucket/video.mp4",
      "credentials": "s3_production"
    }
  ]
}

    Ver también: url y base_url

    parallel_upload_limit

    parallel_upload_limit:Integer

    Versiones de API: V1, V2

    Predeterminado: 30 para S3, 10 para todos los demás destinos.

    Valores válidos: Un entero entre 1 y 30

    Tipos de trabajo compatibles: vod

    Ejemplo: 5

    Descripción:

    El número máximo de cargas simultáneas realizadas al cargar salidas multiparte, principalmente segmentos HLS.

    parallel_upload_limit debe configurarse para salidas individuales, no para todo el trabajo.

     

    Esto puede acelerar los tiempos de transferencia, dependiendo del ancho de banda del servidor remoto. Tenga en cuenta que más conexiones pueden colocar una carga más pesada en el servidor. Si tiene problemas con los tiempos de espera de carga o desea evitar que Zencoder use demasiado ancho de banda al cargar archivos, configure esto en 1.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "parallel_upload_limit": 5,
      "url": "s3://my-output-bucket/final.mp4",
      "credentials": "s3_production"
    },
    {
      "parallel_upload_limit": 5,
      "device_profile": "mobile/advanced",
      "url": "s3://my-output-bucket/mobile.mp4",
      "credentials": "s3_production"
    }
  ]
}

    headers

    headers:Hash

    Versiones de API: V1, V2

    Padres: salidas O miniaturas

    Tipos de trabajo compatibles: Todos

    Ejemplo:{ "Content-Type": "binary/octet-stream" }

    Descripción:

    encabezados HTTP para enviar con su archivo cuando lo subamos. Esta función se admite actualmente cuando se utilizan archivos S3, GCS y Cloud.

    Zencoder admite la configuración de un subconjunto limitado de estos encabezados: Cache-Control , Content-Disposition , Content-Encoding , Content-Type , Expires , x-amz-acl , x-amz-storage-class , x-amz-server-side-encryption , x-amz-meta-* , x-goog-acl y x-goog-meta-*.

    Notas:
    • Para utilizar la función Standard_IA de S3, deberá establecer el x-amz-storage-class encabezado en el valor apropiado. Estos valores cambian y son controlados por AWS y Google, por lo que tendrá que obtener los valores correctos de ellos.
    • Los encabezados «goog» son compatibles con las URL de salida de GCS, pero no para las URL en S3.

    Los encabezados se ignorarán cuando no se especifique una ubicación de salida.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "url": "s3://my-bucket/file.mp4",
      "headers": {
        "Content-Type": "binary/octet-stream",
        "x-amz-acl": "public-read-write"
      }
    }
  ]
}
