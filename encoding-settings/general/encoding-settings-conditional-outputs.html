min_size

min_size:String

Versiones de API: V2

Padre: condicionales

Tipos de trabajo compatibles: Todas

Ejemplo:

  • 640x480

Descripción:

Si las dimensiones de la fuente son más pequeñas que el tamaño solicitado, se omitirá la salida.

Las dimensiones se calculan multiplicando la altura y el ancho de la fuente y comparándolas con el producto de las dimensiones proporcionadas.

Nota: Debido a la naturaleza de los metadatos del video y las inconsistencias entre los tipos de video, algunos archivos de origen pueden tener sus dimensiones leídas incorrectamente durante la inspección, dando como resultado que las condiciones se apliquen o ignoren incorrectamente.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "min_size": "640x480"
      }
    }
  ]
}

min_duration

min_duration:Integer

Versiones de API: V2

Padre: condicionales

Valores válidos: Un entero positivo

Tipos de trabajo compatibles: Todas

Ejemplo:

  • 300

Descripción:

Si la duración de la fuente es más corta que la duración solicitada, se omitirá la salida.

La duración debe expresarse en segundos. Para omitir una salida si la fuente tiene menos de 5 minutos, use '300'.

Nota: Debido a la naturaleza de los metadatos del video y las inconsistencias entre los tipos de video, es posible que la duración de algunos archivos de origen se lea incorrectamente durante la inspección, lo que da como resultado que las condiciones se apliquen o ignoren incorrectamente.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "min_duration": "300"
      }
    }
  ]
}

min_audio_bitrate

min_audio_bitrate:Integer

Versiones de API: V2

Padre: condicionales

Valores válidos: Un entero positivo

Tipos de trabajo compatibles: Todas

Ejemplo:

  • 128

Descripción:

Si la tasa de bits de audio de la fuente es menor que la tasa de bits especificada (en kbps), se omitirá la salida.

Los archivos de origen sin una pista de audio o sin una tasa de bits de audio reconocible ignorarán esta configuración. (Ver require_audio para más opciones.) Para omitir una salida si la tasa de bits de audio de la fuente es inferior a 128 Kbps, utilice 128.

Nota: Debido a la naturaleza de los metadatos de audio y las inconsistencias entre los tipos de audio, es posible que algunos archivos de origen tengan sus tasas de bits leídas incorrectamente durante la inspección, lo que da como resultado que las condiciones se apliquen o ignoren incorrectamente.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "min_audio_bitrate": 128
      }
    }
  ]
}

Ver también: max_audio_bitrate y require_audio

require_audio

require_audio:Boolean

Versiones de API: V2

Padre: condicionales

Defecto: falso

Valores válidos: verdadero o falso

Tipos de trabajo compatibles: Todas

Ejemplo: true

Descripción:

Si el archivo de origen no tiene una pista de audio, se omitirá la salida.

Para omitir una salida si la fuente no tiene una pista de audio, establezca esta opción en verdadero.

Nota: Debido a la naturaleza de los metadatos de los medios y las inconsistencias entre los tipos de medios, algunos archivos de origen pueden tener sus metadatos leídos incorrectamente durante la inspección, lo que da como resultado que las condiciones se apliquen o ignoren incorrectamente.


  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "require_audio": true,
        "min_audio_bitrate": 64
      }
    }
  ]
}
