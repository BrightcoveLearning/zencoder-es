    Configuración de CAE

    Este tema cubre la configuración de codificación consciente de contexto (CAE) para Zencoder.

    Resumen

    La codificación consciente de contexto crea varias copias del vídeo de origen basándose en las directrices que proporcione, pero también en función de las características del vídeo en sí. Esto ayuda a evitar el costo de transcodificación y almacenamiento de sus vídeos mediante la creación de representaciones que serán esencialmente inútiles, dadas las características de la fuente.

    Consulte la Guía CAE para obtener información adicional y recomendaciones sobre la configuración.

    video_codecs

    video_codecs:Array

    Versiones de API: V2

    Padres:dynamic_profile_options

    Valores de elemento de matriz válidos:,h264hevc

    Valor predeterminado: ninguno

    Tipos de trabajo compatibles: Vídeo a la carta

    Ejemplo:["h264", "hevc"]

    Descripción:

    Para salidas de códecs mixtos, una matriz de los códecs para los que crear copias. Actualmente, los valores de elemento permitidos son h264 y hevc.

    {
    "dynamic_profile_options": {
      "max_renditions": 6,
      "video_codecs" : ["h264", "hevc"'],
      "video_codec_options": {
        "h264": {
          "min_renditions": 2,
          "max_resolution": { "width": 1280, "height": 720 }
        },
        "hevc": {
          "min_renditions": 2,
          "max_resolution": { "width": 1920, "height": 1080 }
        }
      }
    }
  }

    video_codec_options

    video_codecs:Object

    Versiones de API: V2

    Padres:dynamic_profile_options

    Valor predeterminado: ninguno

    Tipos de trabajo compatibles: Vídeo a la carta

    Ejemplo:

    "video_codec_options": {
  "h264": {
    "min_renditions": 2,
    "max_resolution": { "width": 1280, "height": 720 }
  },
  "hevc": {
    "min_renditions": 2,
    "max_resolution": { "width": 1920, "height": 1080 }
  }
}

    Descripción:

    Para salidas de códec mixto, un objeto que especifica las opciones para cada códec. Cada propiedad es un objeto con el nombre h264 o hevc , sus propiedades son el conjunto de restricciones para las copias de ese códec. Las propiedades permitidas para estos objetos son:

    • min_renditions requerido
    • min_bitrate
    • max_bitrate
    • max_first_rendition_bitrate
    • min_resolution
    • max_resolution
    • max_first_rendition_resolution
    Nota: estos ajustes también se utilizan a nivel global. Cuando se utilizan configuraciones tanto a nivel global como por códec, los valores deben ser compatibles. Por ejemplo, max_renditions establecido en el nivel global debe ser mayor o igual que la suma de los valores min_rendition establecidos en el nivel por códec.
    {
    "dynamic_profile_options": {
      "max_renditions": 6,
      "video_codecs" : ["h264", "hevc"'],
      "video_codec_options": {
        "h264": {
          "min_renditions": 2,
          "max_resolution": { "width": 1280, "height": 720 }
        },
        "hevc": {
          "min_renditions": 2,
          "max_resolution": { "width": 1920, "height": 1080 }
        }
      }
    }
  }

    quality_rate_tune

    quality_rate_tune:Number

    Versiones de API: V2

    Padres:dynamic_profile_options

    Valores válidos ፦40.0 - 40.0

    Valor predeterminado: 0

    Tipos de trabajo compatibles: Vídeo a la carta

    Ejemplo: 2

    Descripción:

    Al establecer quality_rate_tune un número positivo de 1 a 40, los perfiles CAE tendrán una calidad de vídeo más alta a costa de velocidades de bits más altas. Del mismo modo, dado un número en el rango de -1 a -40, uno puede reducir las velocidades de bits, a costa de disminuir la calidad de video.

    Por ejemplo, con -10 uno puede esperar tasas de bits aproximadamente un 10% más bajas. Con 10, las tasas de bits en general aumentarán alrededor del 10%. Estos números son aproximaciones, ya que CAE para valores positivos de toma precauciones adicionales para garantizar una calidad de vídeo muy alta, y puede haber otros parámetros que limiten las velocidades de bits como la granularidad y el número de copias.quality_rate_tune

    Dado que el contenido típico y la calidad de vídeo deseada frente al ancho de banda varían mucho, puede ser aconsejable experimentar con diferentes valores. 

            {
          "input": "s3://zencodertesting/test.mov",
          "generate_dynamic_profile": true,
          "dynamic_profile_options": {
            "min_renditions": 2,
            "max_renditions": 10,
            "quality_rate_tune": 2
          }
        }

    Additional settings

    A continuación se muestran configuraciones de codificación generales adicionales que se pueden usar para CAE.

    Configuración adicional
    Nombre Tipo Descripción Predeterminado
    speed Entero Ajuste de velocidad de codificación. 3
    one_pass Booleano Forzar la codificación de 1 paso. false
    tuning Cadena Opción de sintonización específica del códecc.  
    start_clip Cadena/Número Cree un subclip del vídeo original, comenzando en un momento determinado. Esto puede ser un código de tiempo con un formato de, o un decimal que expresa el número de segundos en los que empezar.HH:MM:SS.S 0
    clip_length Cadena/Número Cree un subclip de una longitud determinada. Puede ser una duración con un formato de, o un decimal que expresa la longitud del clip en segundos.HH:MM:SS.S duración de origen

    Configuración HDR

    CAE también es compatible con HDR10 y se pueden incluir los siguientes ajustes dynamic_profile_options.

    Configuración adicional
    Nombre Tipo Descripción Predeterminado
    master_display Objeto Hash que contiene los ajustes de volumen de color y luminancia para las salidas HDR. ninguno
    max_content_light_level Entero Nivel máximo de luz para el contenido como un entero liend o candelas por metro cuadrado (cd/m2) - valores enteros. ninguno
    max_frame_average_light_level Entero Nivel de luz medio máximo para liendas o candelas de marcos de contenido por metro cuadrado (cd/m2) - valores enteros. Aunque se permiten valores de hasta 10000, los valores del rango 100-200 generalmente producirán los mejores resultados. ninguno

    Configuración de configuración de vídeo

    Es posible que desee utilizar esta función si CAE no está produciendo configuraciones específicas que debe tener en sus salidas.

    Se trata de una matriz de objetos que describen las resoluciones de vídeo y la configuración de códec que CAE debe producir como parte de su salida (ver el ejemplo anterior). Esta matriz es opcional. Si se omite, se utilizará una matriz de resoluciones predeterminadas, que van desde 192x108 hasta la resolución de contenido como el conjunto de entrada predeterminado para CAE.

    Ajustes de configuración de vídeo
    Nombre Tipo Descripción ¿Obligatorio?
    width Entero Ancho de fotograma de vídeo [píxeles] requerido
    height Entero Altura de fotograma de vídeo [píxeles] requerido
    sample_aspect_ratio Número entero: par de enteros Relación de aspecto de la muestra (p. ej. 1:1, 4:3) opcional
    frame_rate Número Velocidad de fotogramas de vídeo [fps] opcional
    video_codec_profile Cadena perfil de códec de vídeo opcional
    video_codec_level Cadena nivel de códec de vídeo opcional
    video_reference_frames Entero Restricción en el número de marcos de referencia. Predeterminado = 4 (3 para la línea base de H.264). opcional
    video_bframes Entero Restricción en el número de fotogramas B. Predeterminado = 3 (0 para la línea base H.264). opcional
    required Booleano Establecer esto en true garantizará los ajustes de configuración sugeridos en la salida. Valor predeterminado = falso. opcional
